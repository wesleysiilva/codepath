A fase final transforma conhecimento em identidade. Você escolhe uma trilha de interesse, constrói um projeto real do início ao fim, aprende a colaborar com Git e sai com um portfólio público — a evidência concreta de quem você se tornou.

Escolha da Trilha
Para onde ir daqui — 6 direções possíveis
▼

Esta é a decisão mais importante da jornada — e não é irreversível. Você explora as principais áreas da programação, testa um pouquinho de cada uma em aulas de "degustação", e escolhe onde vai aprofundar na fase final. O critério não é "o que dá mais dinheiro" — é "o que faz você querer programar nos fins de semana".

🌐
Desenvolvimento Web
HTML, CSS, JavaScript. Criar sites e aplicações que rodam no navegador. Flask/Django no backend.
🎮
Desenvolvimento de Jogos
Pygame (Python) ou Unity (C#). Física, sprites, colisões, game loop. Projetos com identidade visual.
📊
Ciência de Dados
Pandas, matplotlib, análise exploratória. Transformar dados brutos em insights visuais.
📱
Desenvolvimento Mobile
Flutter (Dart) ou React Native. Apps que rodam no celular com interfaces nativas.
🤖
Automação & Scripts
Selenium, scripts de produtividade, APIs. Automatizar tarefas repetitivas do dia a dia.
🔐
Segurança / Pentest
CTF, análise de vulnerabilidades, redes. Para quem quer entender como sistemas são protegidos (e como são atacados).
🎯
Entregável: Um documento de 1 página escrito por você: qual trilha escolheu, por que, o que espera aprender, e qual seria seu projeto ideal ao final. Essa declaração de intenção será revisitada no final da fase.
🏗️
Projeto Autoral Guiado
Do problema à solução funcionando
▼

O projeto autoral é o coração da Fase 5. É mais complexo que o da Fase 4, envolve a trilha escolhida, e é desenvolvido com autonomia crescente. Você conta com um mentor que faz perguntas em vez de dar respostas e sugere caminhos sem percorrê-los — e você enfrenta e supera as dificuldades reais do desenvolvimento.

Definição e escopoO projeto precisa ser desafiador o suficiente para exigir aprendizado, mas realizável no prazo. Wireframe ou esboço do que será construído. Divisão em milestones com datas.
Pesquisa e aprendizado autônomoVocê aprenderá coisas que ainda não foram ensinadas no curso. Documentação, Stack Overflow, tutoriais — aprender a aprender é a habilidade mais importante desta etapa.
Revisões de código (code review)Você apresenta o código regularmente. Não para corrigi-lo — para receber perguntas: "Por que essa variável tem esse nome?", "O que acontece se o input for vazio aqui?"
Refatoração e versão finalTodo projeto tem uma "versão que funciona" e uma "versão de que você se orgulha". O aluno aprende que refatorar não é um sinal de que errou — é uma parte normal e necessária do desenvolvimento.
🎯
Entregável: Projeto funcionando no GitHub com README detalhado, instruções de instalação e uso, e pelo menos 10 commits com mensagens descritivas que contam a história do desenvolvimento.
🌿
Git Avançado & Colaboração
Branches, pull requests e trabalho em equipe
▼

Na Fase 4, o aluno usou Git como uma forma de salvar versões. Na Fase 5, ele aprende a usar Git como os desenvolvedores profissionais usam — com branches, pull requests e um fluxo de trabalho que permite colaborar sem destruir o código um do outro.

BranchesA ideia de linhas paralelas de desenvolvimento. git branch, git checkout -b, git merge. Por que a branch main deve sempre ter código funcionando.
Conflitos de mergeO que é um conflito e por que ele não é um desastre. Como resolver conflitos manualmente. A habilidade de resolver conflitos de merge é o que diferencia quem realmente usa Git de quem finge usar.
Pull RequestsO fluxo do GitHub: fork, branch, PR, review, merge. Mesmo que o aluno trabalhe sozinho, praticar esse fluxo prepara para colaboração real. Escrever uma boa descrição de PR é uma habilidade profissional.
Contribuição open sourceEncontrar um projeto no GitHub que aceite contribuições iniciantes (label "good first issue"). Fazer o primeiro PR real — mesmo uma correção de typo na documentação já conta e ensina o fluxo completo.
🎯
Entregável: O projeto autoral desenvolvido em branches separadas por feature, com histórico de merge visível. Bônus: uma contribuição real (ou tentativa) a um projeto open source, documentada no README do portfólio.
🎯
Portfólio & Apresentação
Comunicar o que foi construído
▼

Um projeto que não pode ser mostrado é como um livro que ninguém leu. O portfólio é a prova pública do que foi construído — e a habilidade de comunicar tecnicamente é tão importante quanto a habilidade de programar. Esta etapa prepara você para apresentar seu trabalho com clareza e orgulho.

Página de portfólioSite simples (pode ser GitHub Pages) com foto, bio curta, lista de projetos com links e descrições. Não precisa ser elaborado — precisa ser honesto e funcional.
README como carta de apresentaçãoO README do projeto autoral deve responder: o que é, por que foi construído, como instalar, como usar, e o que vem a seguir. Screenshots ou GIF do programa em ação valem muito.
Apresentação técnica oral15 minutos: demonstração ao vivo do projeto, explicação das decisões de design, um problema difícil que foi enfrentado e como foi resolvido, e o que seria feito diferente. Simula uma entrevista técnica.
Perfil GitHub públicoFoto, bio, pinned repositories. O GitHub como currículo técnico — um perfil ativo diz mais sobre um desenvolvedor do que qualquer certificado.
🎯
Entregável: Portfólio online publicado com pelo menos 3 projetos documentados, perfil do GitHub ativo com o projeto autoral em destaque, e apresentação ao vivo de 15 minutos bem executada.
🌱
Comunidade & Próximos Passos
A jornada continua — agora com mapa
▼

Programar solo tem um teto. Programar em comunidade não tem. O módulo final não ensina uma nova tecnologia — ensina como continuar aprendendo, como encontrar pessoas que ajudam, e como transformar curiosidade em carreira. É a preparação para o que vem depois do curso.

Comunidades onlineDiscord de programação, Reddit (r/learnpython, r/webdev), grupos no Telegram e WhatsApp. Como fazer perguntas bem formuladas (quem sabe fazer boas perguntas aprende mais rápido).
Plataformas de práticaLeetCode, HackerRank para algoritmos. freeCodeCamp e The Odin Project para web. Kaggle para dados. Saber onde praticar é metade do caminho.
O que vem depoisDependendo da trilha: cursos aprofundados, faculdade, bootcamp, estágio, freelance, projetos pessoais. Não existe um caminho único — existe o caminho que faz sentido para cada pessoa.
Síndrome do impostorTodo desenvolvedor sente que não sabe o suficiente — inclusive os seniores. Reconhecer isso é o primeiro passo para não deixá-la paralisar. O antídoto: continuar construindo coisas.
🎯
Entregável final da jornada: Um parágrafo escrito por você descrevendo para onde quer ir a partir daqui — qual área quer explorar, qual projeto quer construir, qual comunidade quer participar. Essa é a declaração de intenção de um desenvolvedor. A jornada não termina aqui.
