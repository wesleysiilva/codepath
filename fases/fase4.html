Ambiente e Primeiros Passos
Configurar, instalar, escrever o primeiro programa
▼

Antes de escrever Python, você precisa de um ambiente funcional. Configurar o ambiente é uma das partes mais frustrantes para iniciantes — e também uma das mais importantes, porque desenvolvedores reais passam tempo real configurando ambientes. Normalizar isso desde o início é fundamental.

Instalar Pythonpython.org, diferença entre Python 2 e 3 (sempre 3). Verificar instalação no terminal com python --version. Entender o PATH: por que o terminal não acha o Python quando mal instalado.
VS Code como editorInstalação e extensão Python. Sintaxe highlight, autocomplete, terminal integrado. Por que um bom editor importa. Alternativa no browser: Google Colab para quem não quer instalar nada.
Hello, World!O primeiro programa. print("Hello, World!"). Por que esse ritual existe em todas as linguagens. Rodar no terminal vs. rodar pelo editor.
Ambientes virtuais (venv)Por que isolar projetos evita conflito de pacotes. python -m venv venv, ativar e desativar. Boa prática desde o primeiro dia.
🎯
Entregável: Python instalado, VS Code configurado com extensão Python, e um script ola_mundo.py rodando no terminal. Você consegue modificar a mensagem e rodar novamente sem ajuda.

Variáveis, Tipos e Operadores
Os blocos fundamentais de qualquer programa
▼

Aqui a lógica da Fase 3 se concretiza em código Python. Os conceitos de variáveis e tipos já foram internalizados — agora é questão de aprender a sintaxe específica do Python, que é propositalmente próxima do inglês e do pseudocódigo.

Tipos básicosint, float, str, bool. Python é dinamicamente tipado — a variável não precisa declarar tipo. type() para inspecionar. Conversão entre tipos: int("42"), str(100).
Operadores aritméticos+, -, *, /, //, %, **. Divisão inteira (//): "quantas vezes cabe" sem o resto. Módulo (%): o resto da divisão — você aprende que isso é útil para verificar paridade e muito mais.
Strings em profundidadef-strings: f"Olá, {nome}!". Métodos essenciais: .upper(), .split(), .strip(), .replace(). Indexação e slicing: texto[0], texto[1:5].
Input do usuárioinput() sempre retorna string — a armadilha mais comum para iniciantes. Converter para int antes de operar matematicamente. O programa passa a ser interativo.
🎯
Entregável: Calculadora de IMC que pede nome, peso e altura via input(), calcula o IMC, exibe a categoria e uma mensagem personalizada usando f-string. Tudo em menos de 15 linhas.

Condicionais, Laços e Lógica
Controlar o fluxo do programa
▼

A lógica da Fase 3 (if/else, while, for) agora tem sintaxe Python. A diferença mais marcante: Python usa indentação para definir blocos — o que parece estranho no começo, mas rapidamente se torna natural e elimina um erro comum em outras linguagens.

if / elif / elseA indentação como delimitador de bloco — e o TypeError que acontece quando se ignora isso. Condições compostas com and, or, not.
for e range()for i in range(10). Iterar sobre listas e strings diretamente: for letra in palavra. enumerate() para ter índice e valor ao mesmo tempo.
whileLoop controlado por condição. break e continue. O jogo de adivinhação como projeto prático: while + input + comparação + contador de tentativas.
Listas e operaçõesCriar, acessar por índice, adicionar (.append()), remover (.remove()), ordenar (.sort()). Iterar sobre lista com for. List comprehension como prévia.
🎯
Entregável: Jogo de adivinhação completo: o programa sorteia um número aleatório (usando random.randint), o usuário chuta, recebe "maior" ou "menor" até acertar, e ao final exibe quantas tentativas foram usadas.

Funções e Organização do Código
Reutilizar, modularizar e escalar
▼

Funções transformam um script de procedimento linear em um programa modular. Esta é uma das transições mais importantes: você para de "despejar código" e começa a organizar. A capacidade de escrever boas funções é o sinal mais claro de que alguém está progredindo como programador.

def, parâmetros e returnDefinir e chamar funções. A diferença entre parâmetro e argumento. Retornar vs. imprimir: um erro conceitual muito comum que precisa ser corrigido cedo.
Escopo de variáveisLocal vs. global. Por que variáveis dentro de funções não existem fora delas. A armadilha do global e por que evitá-la na maioria dos casos.
Dicionários em profundidade{chave: valor}. Acessar, adicionar, remover. .get() para acesso seguro. Iterar com .items(), .keys(), .values(). Lista de dicionários para representar registros.
Tratamento de errostry / except. Tipos de exceção: ValueError, TypeError, KeyError. Não silenciar erros com except: pass. Mensagens de erro úteis para o usuário.
🎯
Entregável: Sistema de cadastro de alunos com menu interativo, usando lista de dicionários para armazenar os dados, funções separadas para cada operação (adicionar, listar, buscar, remover) e tratamento de erros para inputs inválidos.

Arquivos, Bibliotecas e Git
Persistência, ecossistema e versionamento
▼

Programas que perdem dados ao fechar são limitados. Programas que usam só código próprio são reinventar a roda. Programas sem controle de versão são um perigo. Este módulo resolve os três problemas e conecta o aluno ao ecossistema real do desenvolvimento Python.

Leitura e escrita de arquivosopen() com with. Modos: r, w, a. Ler linha por linha. JSON como formato de dados estruturado: json.load() e json.dump(). O cadastro do módulo anterior agora salva em arquivo.
Bibliotecas padrãorandom, datetime, os, math. A ideia de que Python vem "com bateria incluída". Como ler a documentação oficial.
pip e bibliotecas externaspip install e requirements.txt. PyPI como repositório central. Instalar e usar requests para fazer chamadas HTTP em Python.
Git básicoinit, add, commit, push. O que é um repositório. GitHub como hospedagem. .gitignore. O hábito de fazer commit com mensagens descritivas.
🎯
Entregável: O sistema de cadastro do módulo anterior com persistência em JSON + README escrito pelo aluno explicando o projeto + repositório no GitHub com pelo menos 5 commits com mensagens descritivas.

Projeto Autoral
Um programa real, de escolha do aluno
▼

Este é o módulo mais importante da fase — e talvez de todo o curso até aqui. Você escolhe um problema que quer resolver, projeta a solução, implementa, documenta e apresenta. Não há roteiro a seguir. Você conta com orientação, não com respostas prontas. Essa transição muda tudo.

Escolha do problemaVocê propõe um projeto que resolve algo real para você. Exemplos: gerenciador de tarefas, quiz de perguntas, calculadora de despesas, bot para consultar API de CEP, gerador de senhas seguras.
Design antes do códigoFluxograma do programa. Quais funções existirão. Quais dados serão guardados e em qual estrutura. Você escreve isso antes de abrir o VS Code — prática que distingue programadores experientes de iniciantes.
Desenvolvimento iterativoVersão 1 funciona minimamente. Versão 2 adiciona uma feature. Versão 3 refatora o que ficou feio. Cada versão é um commit no Git. Desenvolvimento real raramente é linear.
Documentação e apresentaçãoREADME que explica o que o programa faz, como instalar e como usar. Apresentação de 5 minutos: o que foi construído, quais desafios surgiram, o que você faria diferente numa versão 2.
🎯
Entregável final da fase: Programa Python funcional no GitHub, com funções, persistência em arquivo, tratamento de erros, README bem escrito e apresentação ao vivo de 5 minutos. O critério principal: você consegue explicar cada decisão que tomou.
