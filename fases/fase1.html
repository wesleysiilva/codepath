Arquitetura de Von Neumann
O modelo que define computadores modernos
▼

John von Neumann, em 1945, descreveu uma arquitetura que define praticamente todos os computadores modernos até hoje. Entender esse modelo é entender por que o computador funciona como funciona — e por que programas são o que são. Este é um dos pilares teóricos mais importantes do curso.

Memória unificadaA ideia revolucionária: dados e instruções do programa ficam na mesma memória. Antes de Von Neumann, os programas eram "queimados" no hardware (cabos e chaves). Isso permitiu que softwares fossem criados e modificados facilmente.
→
Unidade de Controle (UC)O "maestro" que busca instruções na memória, decodifica o que elas significam e coordena os outros componentes para executá-las. É ela que diz "agora some isso, agora guarda aquilo".
→
ULA — Unidade Lógica e AritméticaA "calculadora" do processador. Realiza todas as operações matemáticas (soma, subtração, multiplicação) e lógicas (AND, OR, NOT, comparações). Tudo que o computador "pensa" passa pela ULA.
→
Ciclo Fetch-Decode-ExecuteO loop infinito do processador: buscar instrução na memória → decodificar o que significa → executar a operação → repetir. Um processador moderno faz bilhões de ciclos por segundo.
→
RegistradoresMemória ultra-rápida dentro do processador (poucos bytes). Variáveis do nível mais baixo possível. O Contador de Programa (PC) guarda o endereço da próxima instrução a executar.
→
Barramento (Bus)Os "corredores" que conectam CPU, memória e I/O. Barramento de dados, de endereço e de controle — cada um com sua função. A largura do barramento define quantos bits viajam de uma vez.
→
🎯
Entregável: Você desenha o diagrama de Von Neumann do zero, sem consultar, e explica em voz alta o que cada componente faz usando uma analogia própria (crie a sua — não repita a que viu no conteúdo). Isso confirma internalização real, não memorização.

Hardware & Componentes
O que há dentro do computador
▼

O ponto de partida é a desmistificação: o computador não é uma caixa preta mágica. É um conjunto de peças físicas com funções muito específicas, trabalhando em harmonia. Antes de escrever qualquer linha de código, você precisa saber o que está acontecendo dentro da máquina que vai executar esse código.

CPU — o cérebroNúcleos, clock, pipeline. Como o processador executa instruções uma a uma (ou várias em paralelo). Analogia com um cozinheiro seguindo uma receita.
→
Memória RAMVolátil e rápida. A "bancada de trabalho" — existe para acesso imediato, não para guardar permanentemente. Por que reiniciar o PC perde dados abertos.
→
Armazenamento: HD e SSDPersistente e mais lento. A diferença de velocidade entre HD (mecânico) e SSD (eletrônico) e por que isso importa no dia a dia do desenvolvedor.
→
GPU e placa-mãeA GPU como processador especializado em paralelismo (e por que a IA adora GPUs). A placa-mãe como o "esqueleto" que conecta tudo.
→
Periféricos I/ODispositivos de entrada (teclado, mouse, câmera) e saída (monitor, caixas de som). A cadeia completa de uma ação do usuário até a resposta na tela.
→
Atividade práticaDesmontar (virtualmente, com simuladores, ou fisicamente com um PC antigo) e identificar cada componente. Criar um mapa visual da máquina.
→
🎯
Entregável: Diagrama desenhado à mão ou digital com todas as peças do computador, suas funções e como se comunicam. Você deve conseguir explicar verbalmente o que cada componente faz.

Sistemas Operacionais
O software que gerencia tudo
▼

O sistema operacional é o intermediário entre o hardware e os programas. Sem entendê-lo, você ficará perdido ao instalar ferramentas, resolver permissões ou entender por que um código roda diferente no Windows e no Linux. Esta etapa desmistifica o que acontece "debaixo do capô" do SO.

O que é um SOA camada de abstração entre o hardware e o usuário. Por que precisamos de um sistema operacional — sem ele, cada programa precisaria falar diretamente com o hardware.
→
Windows, Linux e macOSDiferenças filosóficas (licença aberta vs. fechada), mercados de uso (servidores adoram Linux), e por que desenvolvedores frequentemente preferem sistemas baseados em Unix.
→
Processos e memóriaO que é um processo? Como o SO gerencia vários programas rodando ao mesmo tempo (multitasking). O que acontece quando você abre o Spotify e o Chrome juntos.
→
Sistema de arquivosHierarquia de pastas e arquivos. Permissões (quem pode ler, escrever, executar). Caminhos absolutos e relativos — conceito crucial para programação.
→
🎯
Entregável: Você abre o Gerenciador de Tarefas (Windows) ou Monitor de Atividade (Mac) / htop (Linux) e explica o que está vendo — quais processos existem, quanto de memória cada um usa, e o que acontece ao abrir um novo programa.

Representação de Dados
Como o computador entende o mundo
▼

O computador só conhece dois estados: ligado e desligado, 1 e 0. Mas com isso ele representa números, letras, imagens, sons e vídeos. Entender como essa tradução funciona é fundamental para compreender por que bugs de overflow existem, por que arquivos têm tamanhos, e o que é encoding de texto na prática.

Sistema binárioBase 2 vs. base 10. Como converter entre decimal e binário. Por que o computador usa binário (estados físicos de transistores). Bit e byte.
→
HexadecimalBase 16 como atalho para o binário. Cores no CSS (#FF5733) são hexadecimais. Endereços de memória também. Você aprende a "ler" hex intuitivamente.
→
Números inteiros e negativosComplemento de dois — como representar números negativos em binário. Overflow: o que acontece quando um número ultrapassa o limite de bits disponíveis.
→
Texto: ASCII e Unicode/UTF-8Como letras viram números. ASCII (128 caracteres) e seus limites. Unicode como solução global. Por que "encoding" importa e o que causa aquele caractere estranho "Ã©" em vez de "é".
→
Imagens e pixelsRGB como três números (0–255). Uma imagem como uma grade de pixels. O que é resolução. Por que uma foto de 12 megapixels ocupa tanto espaço.
→
CompressãoIdeia geral de compressão com e sem perdas. Por que um MP3 soa parecido com o WAV original mas ocupa 10x menos. JPEG vs PNG para imagens.
→
🎯
Entregável: Mini-tabela mostrando como seu nome é representado em ASCII/Unicode, com os valores decimais, binários e hexadecimais de cada letra. Pode ser feito em papel ou planilha.

Linha de Comando
Falar diretamente com o sistema operacional
▼

A interface gráfica é uma camada de conveniência. O terminal é onde o controle de verdade acontece. Todo desenvolvedor usa o terminal diariamente — para rodar programas, instalar pacotes, navegar em pastas, usar Git. Esta etapa tira o medo do terminal e transforma ele numa ferramenta natural.

O que é o terminal / shellA diferença entre terminal, shell e prompt. Bash, Zsh, PowerShell. Por que o terminal parece intimidador e por que isso passa rapidamente com prática.
→
Comandos essenciais de navegaçãopwd, ls/dir, cd, mkdir, rm/del. Criar, mover e apagar pastas e arquivos. Caminhos absolutos vs. relativos no terminal.
→
Manipulação de arquivoscat, echo, cp, mv. Redirecionar saída para um arquivo (> e >>). Pipes (|) — encadear comandos para criar fluxos de processamento.
→
Permissões e variáveis de ambientechmod (Linux/Mac). PATH: por que digitar python no terminal funciona (ou não). Como adicionar algo ao PATH. sudo e quando usá-lo com cuidado.
→
🎯
Entregável: Você navegando pelo sistema de arquivos usando apenas o terminal, cria uma estrutura de pastas de um projeto fictício, cria um arquivo de texto sem sair do terminal, e move esse arquivo para outra pasta. Tudo sem tocar no mouse.
