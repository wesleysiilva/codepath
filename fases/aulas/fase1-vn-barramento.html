Fase 1 · Arquitetura de Von Neumann

Barramento:
a rodovia do computador

O barramento é o conjunto de fios que conecta CPU, memória e dispositivos de I/O. Todo dado que flui dentro do computador — uma instrução buscada, um pixel renderizado, um arquivo lido — trafega por algum barramento.

⏱ ~12 min
📶 Iniciante
Módulo 1 de 5

O que é um barramento?

Na arquitetura de Von Neumann, CPU, memória e dispositivos de I/O são componentes separados. Para que eles troquem informação, precisam de um canal de comunicação. Esse canal é o barramento (bus) — um conjunto de trilhas condutoras (fisicamente, trilhas de cobre na placa-mãe) por onde os sinais elétricos trafegam.

Analogia da rodovia: Pense nos três barramentos como três rodovias paralelas: uma para dizer o destino (endereço), outra para transportar a carga (dados), e uma terceira para os sinais de trânsito (controle: semáforo, sentido, velocidade).

Os três barramentos clássicos

Barramento de Endereços

Onde?

Trafega o endereço do local de memória (ou dispositivo) a ser acessado. É unidirecional — somente a CPU envia endereços. A largura define o espaço endereçável: 32 bits = até 4 GB de RAM; 64 bits = 16 exabytes.

Barramento de Dados

O quê?

Trafega os dados reais: bytes lidos da memória, valores escritos em registradores, pixels enviados à GPU. É bidirecional. A largura (8, 16, 32, 64 bits) define quantos bits são transferidos por ciclo.

Barramento de Controle

Como?

Trafega os sinais que coordenam a operação: clock, leitura/escrita, confirmação de recebimento (ACK), requisições de interrupção (IRQ). A UC usa este barramento para orquestrar toda comunicação.

O gargalo de Von Neumann

A arquitetura de Von Neumann tem uma limitação fundamental: dados e instruções compartilham o mesmo barramento. Enquanto uma instrução está sendo buscada (fetch), dados não podem ser transferidos — e vice-versa. Isso cria um gargalo.

Esse problema é tão clássico que ganhou nome próprio: Von Neumann bottleneck. Ele é uma das principais razões pelas quais a CPU frequentemente fica esperando por dados enquanto poderia estar processando — um estado chamado de memory stall.

🚧
Como a indústria mitiga isso: Cache L1 separado por instrução e dado (arquitetura Harvard modificada), múltiplos canais de memória (dual-channel DDR aumenta largura de banda), prefetching preditivo (a CPU busca dados antes de precisar deles), e barramentos especializados para I/O de alta velocidade (PCIe para GPU e NVMe).

Evolução: dos barramentos compartilhados ao PCIe

Nos primeiros PCs, todos os dispositivos compartilhavam um único barramento lento (ISA, PCI). Com o tempo, a arquitetura evoluiu para barramentos ponto-a-ponto dedicados:

  1. ISA
    Industry Standard Architecture (anos 80–90) — barramento de 8 ou 16 bits, compartilhado por todos os dispositivos. Lento e com conflitos frequentes.
  2. PCI
    Peripheral Component Interconnect (anos 90–2000) — 32 ou 64 bits, mais rápido, ainda compartilhado. Todos os dispositivos "disputavam" a mesma via.
  3. AGP
    Accelerated Graphics Port (final dos 90) — barramento dedicado para GPU. Primeiro passo para conexões ponto-a-ponto especializadas.
  4. PCIe
    PCI Express (2003–hoje) — cada dispositivo tem sua própria via serial de alta velocidade. PCIe 5.0 × 16 (para GPU) transfere até 64 GB/s. É o padrão moderno.
Memória RAM

DDR via IMC

A RAM moderna se conecta diretamente ao Memory Controller integrado na CPU (IMC). DDR5 em dual-channel oferece até ~100 GB/s de largura de banda. Sem passar pela placa-mãe.

SSD NVMe

PCIe direto

SSDs NVMe modernos usam PCIe × 4 e atingem até 7 GB/s de leitura sequencial. Comparado aos 550 MB/s máximos do SATA, é 12× mais rápido — graças ao barramento dedicado.

USB / Thunderbolt

I/O externo

Periféricos externos usam protocolos como USB 3.2 (20 Gbps) ou Thunderbolt 4 (40 Gbps). Internamente, esses controladores se conectam à CPU via PCIe ou diretamente no SoC.

Chipset / PCH

O hub dos barramentos

O Platform Controller Hub (PCH) gerencia dispositivos de menor velocidade (USB, SATA, áudio). Conecta-se à CPU por uma via DMI dedicada, liberando o processador para os dispositivos de alta velocidade.

Por que isso importa para você, programador?

A maioria dos programadores nunca manipula barramentos diretamente. Mas entender sua existência explica comportamentos que você vai encontrar:

Largura de banda vs. latência

São coisas diferentes

Um SSD NVMe tem alta largura de banda (GB/s) mas ainda tem latência (~100 µs vs. ~100 ns da RAM). Ler um arquivo pequeno é "caro" não pela velocidade do barramento, mas pelo tempo de resposta. Isso explica por que I/O em disco é mais lento que RAM mesmo em SSDs rápidos.

Cache e localidade

Acessar dados próximos é mais rápido

Quando a CPU precisa de dados, ela busca um bloco de ~64 bytes (cache line) do barramento. Acessar elementos sequencialmente (arrays) é muito mais eficiente do que acessos aleatórios (linked lists) — você aproveita o dado que já veio pelo barramento.

I/O bound vs. CPU bound

Onde está o gargalo?

Programas que passam mais tempo esperando dados (disco, rede, RAM) são I/O bound. Os que passam mais tempo calculando são CPU bound. Essa distinção determina se usar mais threads vai ajudar ou não. E sua origem é o barramento.

Gargalos de hardware

Diagnóstico de performance

Ferramentas como perf, vtune ou o Task Manager mostram uso de CPU, memória e disco separadamente. Se a CPU está ociosa mas o disco está 100%, o gargalo é no barramento de I/O — não no processador.

Termos-chave desta aula

Barramento (Bus) Conjunto de trilhas elétricas que conecta CPU, memória e dispositivos de I/O, permitindo transferência de dados.
Barramento de endereços Via unidirecional (CPU → memória) que carrega o endereço do local a ser lido ou escrito. Sua largura define o máximo de RAM endereçável.
Barramento de dados Via bidirecional que trafega os dados reais entre CPU, memória e I/O. Largura típica: 64 bits.
Barramento de controle Via que trafega sinais de coordenação: clock, leitura/escrita, interrupções (IRQ), confirmações (ACK).
Von Neumann bottleneck Gargalo causado pelo barramento compartilhado entre dados e instruções, que impede a CPU de buscar instruções e dados simultaneamente.
PCIe PCI Express. Barramento serial ponto-a-ponto moderno. Usado por GPUs, SSDs NVMe e placas de rede de alta velocidade.
Largura de banda Quantidade máxima de dados transferidos por segundo (GB/s). Diferente de latência, que mede o tempo de resposta de uma requisição.
Cache line Bloco de ~64 bytes transferido do barramento para o cache de uma vez. Acessos sequenciais aproveitam a cache line inteira; acessos aleatórios desperdiçam.
