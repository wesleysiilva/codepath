Permissões e variáveis
de ambiente

chmod, sudo, PATH e variáveis de ambiente são conceitos que aparecem cedo para qualquer desenvolvedor — ao instalar ferramentas, rodar scripts ou configurar ambientes de desenvolvimento. Esta aula desmistifica cada um deles.

⏱ ~15 min
📶 Iniciante
Módulo 5 de 5

chmod: mudar permissões de arquivo

O comando chmod (change mode) muda quem pode ler, escrever ou executar um arquivo. Você já aprendeu a teoria das permissões — agora veja como aplicá-las:

bash — chmod
# Ver permissões atuais
$ ls -l script.py
-rw-r--r-- 1 wesley staff 1024 Apr 19 script.py
# rw- r-- r-- : dono lê/escreve, outros só leem. Não executa.

# Tornar executável pelo dono
$ chmod u+x script.py
$ ls -l script.py
-rwxr--r-- 1 wesley staff 1024 Apr 19 script.py

# Forma numérica: 755 = rwxr-xr-x
$ chmod 755 script.py

# Executar o script diretamente
$ ./script.py
Hello World!
🔢
chmod em octal — os mais usados:
755 = rwxr-xr-x — scripts, executáveis (todos executam, só dono escreve)
644 = rw-r--r-- — arquivos normais (dono lê/escreve, outros só leem)
600 = rw------- — arquivos privados (chaves SSH, .env com senhas)
777 = rwxrwxrwx — todos têm tudo — evite em produção!

sudo: executar como administrador

Algumas operações requerem privilégios de administrador (root no Linux/Mac, Administrador no Windows). O comando sudo (super user do) executa um comando com privilégios elevados.

bash — sudo
# Sem sudo: instalar pacote do sistema falha
$ apt install nginx
E: Could not open lock file ... - open (13: Permission denied)

# Com sudo: solicita senha, depois executa como root
$ sudo apt install nginx
[sudo] password for wesley:
Reading package lists... Done

# Ver quem é o usuário atual
$ whoami
wesley
$ sudo whoami
root
⚠️
Use sudo com cuidado: Como root, você pode fazer qualquer coisa — incluindo apagar arquivos críticos do sistema. Nunca execute sudo rm -rf / ou comandos que você não entende. Regra prática: use sudo só quando necessário (instalar pacotes do sistema, configurar firewall, acessar logs do sistema). Para instalar pacotes Python (pip install), prefira ambientes virtuais — sem sudo.

Variáveis de ambiente

Variáveis de ambiente são pares chave=valor que o SO disponibiliza para todos os processos. Programas as usam para configuração: onde encontrar executáveis, tokens de API, ambiente de execução (desenvolvimento vs. produção).

bash — variáveis de ambiente
# Ver todas as variáveis de ambiente
$ env
HOME=/home/wesley
USER=wesley
PATH=/usr/local/bin:/usr/bin:/bin:...
LANG=pt_BR.UTF-8

# Ver uma variável específica
$ echo $HOME
/home/wesley

# Definir variável temporária (só nesta sessão)
$ export MINHA_VAR="valor_aqui"
$ echo $MINHA_VAR
valor_aqui

# Acessar em Python
$ python3 -c "import os; print(os.environ['MINHA_VAR'])"
valor_aqui

PATH: por que "python" funciona no terminal

Quando você digita python no terminal, como o shell sabe onde está o programa Python? Ele busca nos diretórios listados na variável PATH, um por um, até encontrar um executável com esse nome.

PATH = lista de diretórios separados por :
📁/usr/local/binprogramas instalados pelo usuário
📁/usr/binprogramas do sistema
📁/binutilitários essenciais (ls, cp, bash...)
📁/home/wesley/.local/binprogramas do usuário (pip instala aqui)
bash — PATH
# Ver o PATH atual
$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/home/wesley/.local/bin

# Onde está o python?
$ which python3
/usr/bin/python3

# Adicionar pasta ao PATH permanentemente (no ~/.bashrc ou ~/.zshrc)
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc # recarregar configurações
💡
"command not found" — o diagnóstico: Quando o terminal diz command not found: python, significa que o executável não está em nenhum diretório do PATH. Soluções: (1) instalar o programa, (2) adicionar o diretório onde ele está ao PATH, ou (3) usar o caminho completo (/usr/bin/python3 em vez de só python3).

Boas práticas com variáveis de ambiente

Segredos

Nunca no código-fonte

Senhas, tokens de API e chaves de banco de dados nunca devem ficar no código. Use variáveis de ambiente: API_KEY=xxx python app.py ou armazene em arquivos .env (nunca no git).

Arquivo .env

Padrão para projetos

Crie um arquivo .env com as variáveis do projeto. Use python-dotenv para carregá-las automaticamente. Adicione .env ao .gitignore — jamais suba segredos para o GitHub.

Ambientes

dev vs. produção

ENVIRONMENT=development ou ENVIRONMENT=production permite que o mesmo código se comporte diferente. Em dev, exibe logs detalhados e usa banco local. Em prod, mínimo de logs e banco remoto.

Termos-chave desta aula

chmod Change mode. Modifica as permissões de leitura, escrita e execução de arquivos e diretórios Unix.
sudo Super User DO. Executa um comando com privilégios de root. Necessário para instalar pacotes do sistema, modificar arquivos protegidos.
Variável de ambiente Par chave=valor disponível para todos os processos. Usada para configuração, senhas, tokens de API e controle de ambiente (dev/prod).
PATH Variável de ambiente que lista os diretórios onde o shell busca executáveis. Quando um comando não é encontrado, o PATH é o lugar a inspecionar.
export Torna uma variável disponível para processos filhos do shell atual. Sem export, a variável existe só no shell onde foi criada.
.bashrc / .zshrc Arquivo de configuração do shell executado ao abrir um terminal. Onde se definem aliases, PATH personalizado e variáveis permanentes.
