Fase 1 · Arquitetura de Von Neumann

Ciclo Fetch-Decode-Execute:
o batimento cardíaco da CPU

A CPU não "pensa" — ela repete o mesmo ciclo bilhões de vezes por segundo: busca uma instrução, decodifica o que ela significa e a executa. Tudo que um computador faz, desde abrir um app até renderizar um jogo, é esse ciclo se repetindo em alta velocidade.

⏱ ~15 min
📶 Iniciante
Módulo 1 de 5

O ciclo em três etapas

O Ciclo Fetch-Decode-Execute (FDE) é o processo fundamental pelo qual a CPU executa qualquer programa. Cada instrução — somar dois números, comparar valores, mover dados — passa por essas três fases antes de produzir qualquer efeito:

Fase 1
Fetch
Buscar a instrução na memória usando o Program Counter
Fase 2
Decode
A UC interpreta o opcode e identifica operandos
Fase 3
Execute
A ULA ou outro componente realiza a operação
💓
O batimento cardíaco da CPU: Um processador a 3 GHz completa 3 bilhões desses ciclos por segundo. Cada ciclo pode executar uma instrução diferente. Quando você digita uma tecla, centenas de milhões de ciclos FDE acontecem antes da letra aparecer na tela.

Fetch: buscar a instrução

Tudo começa com o Program Counter (PC) — um registrador especial que armazena o endereço de memória da próxima instrução a ser executada.

  1. 1
    A UC lê o PC — por exemplo, o valor 0x1000 (endereço 4096).
  2. 2
    O endereço vai ao barramento — a UC coloca o endereço no barramento de endereços e sinaliza "leitura".
  3. 3
    A memória responde — os bits da instrução naquele endereço chegam pelo barramento de dados.
  4. 4
    A instrução entra no IR — os bits são armazenados no Instruction Register.
  5. 5
    O PC é incrementado — automaticamente avança para o próximo endereço (0x1000 → 0x1004 para instruções de 32 bits), pronto para o próximo fetch.
📍
O PC e a execução sequencial: Normalmente o PC avança de instrução em instrução em sequência. Quando você usa um if, while ou chama uma função, o PC pula para outro endereço — é exatamente isso que a instrução de salto (JMP) faz: sobrescreve o PC com um novo valor.

Decode: entender a instrução

Com a instrução no Instruction Register, a Unidade de Controle decodifica os bits. Uma instrução típica tem dois campos principais:

Campo 1

Opcode

Os primeiros bits identificam a operação: é uma soma? Uma comparação? Um desvio? Um acesso à memória? A UC consulta uma tabela interna de opcodes para saber o que fazer.

Campo 2

Operandos

Os bits restantes especificam com o que operar: quais registradores usar, qual endereço de memória acessar, ou um valor imediato (constante embutida na instrução).

Exemplo concreto: em uma arquitetura hipotética de 16 bits, a instrução 0001 0010 0011 0000 poderia ser lida como: opcode 0001 = ADD, registrador destino = R2, registrador fonte = R3. A UC determina: "preciso ativar a ULA no modo soma, com R2 e R3 como entradas, e guardar o resultado em R2."

Execute: realizar a operação

Após decodificar, a UC gera sinais de controle que ativam os componentes corretos para realizar a operação. O que acontece aqui depende totalmente da instrução:

Instrução aritmética

ex: ADD R1, R2

A UC ativa a ULA no modo soma. Os valores de R1 e R2 entram na ULA. O resultado vai de volta a R1. Os flags de status (zero, carry, overflow) são atualizados.

Instrução de memória

ex: LOAD R1, [0x2000]

A UC coloca o endereço 0x2000 no barramento de endereços, sinaliza leitura. A memória retorna o valor. A UC carrega esse valor em R1.

Instrução de salto

ex: JMP 0x3000

A UC sobrescreve o Program Counter com o endereço 0x3000. No próximo ciclo, o fetch buscará a instrução nesse novo endereço. O programa "pula".

Salto condicional

ex: JZ 0x3000

Jump if Zero. A UC verifica o flag ZF nos registradores de status. Se ZF=1, sobrescreve o PC com 0x3000. Se ZF=0, o PC continua normal. Isso é como o if funciona na CPU.

O clock: o metrônomo do ciclo

A CPU possui um clock — um oscilador de cristal que gera pulsos elétricos em frequência constante. Cada pulso sincroniza os componentes da CPU. O FDE é cadenciado por esses pulsos.

F = Fetch · D = Decode · E = Execute — cada ciclo FDE se repete com o clock

A frequência do clock é medida em GHz (gigahertz = bilhões de ciclos por segundo). Um processador a 4 GHz pode completar até 4 bilhões de ciclos por segundo. CPUs modernas com pipelining conseguem até sobrepor etapas de ciclos diferentes para maior eficiência.

🔀
Pipeline — executar múltiplas instruções em paralelo: CPUs modernas não esperam uma instrução completar para começar a próxima. Enquanto a instrução 1 está em Execute, a instrução 2 pode estar em Decode e a instrução 3 em Fetch. É como uma linha de montagem: várias instruções em estágios diferentes ao mesmo tempo. Isso multiplica a taxa de execução sem precisar aumentar o clock.

Do código Python ao ciclo FDE

Quando você escreve x = 5 + 3 em Python, você não vê o FDE — mas ele está lá. Veja a cadeia completa:

  1. 1
    Python interpreta o código — o interpretador Python lê 5 + 3 e gera instruções de bytecode (código intermediário).
  2. 2
    O bytecode é executado — a máquina virtual do Python executa essas instruções, que por sua vez são código de máquina nativo.
  3. 3
    Fetch — a CPU busca a instrução ADD na memória via Program Counter.
  4. 4
    Decode — a UC identifica o opcode ADD e os registradores que contêm 5 e 3.
  5. 5
    Execute — a ULA soma 5 + 3 = 8. O resultado vai para um registrador e eventualmente para a variável x na memória.

Cada linha de código Python pode gerar dezenas ou centenas de instruções de máquina. Cada instrução passa pelo ciclo FDE. O computador não "entende" Python — ele entende apenas bits, endereços e opcodes, repetidos bilhões de vezes por segundo.

Termos-chave desta aula

Fetch Etapa do ciclo FDE onde a CPU busca a instrução na memória usando o Program Counter.
Decode Etapa onde a Unidade de Controle interpreta o opcode e identifica os operandos da instrução.
Execute Etapa onde a operação é realizada — pela ULA, memória ou outros componentes, conforme os sinais de controle da UC.
Program Counter (PC) Registrador que aponta para o endereço da próxima instrução. Incrementado automaticamente após cada fetch; sobrescrito em saltos.
Instruction Register (IR) Registrador que armazena a instrução atual enquanto ela é decodificada e executada.
Clock Oscilador que gera pulsos em frequência constante (ex: 3 GHz = 3 bilhões de pulsos/s), sincronizando todos os componentes da CPU.
Pipeline Técnica onde múltiplas instruções em estágios diferentes do FDE são executadas em paralelo, como uma linha de montagem.
Salto condicional Instrução que sobrescreve o PC somente se uma condição (flag) for verdadeira. É como o if é implementado em hardware.
