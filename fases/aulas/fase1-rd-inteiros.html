Inteiros e negativos:
complemento de dois

Como representar -1 em binário se você só tem 0s e 1s? A solução elegante da computação é o complemento de dois — um truque matemático que permite somar e subtrair com o mesmo circuito. E entender overflow é fundamental para evitar bugs sérios.

⏱ ~12 min
📶 Iniciante
Módulo 4 de 5

Inteiros sem sinal (unsigned)

Um inteiro de 8 bits sem sinal representa valores de 0 a 255 (2⁸ − 1). Com 16 bits: 0 a 65.535. Com 32 bits: 0 a 4.294.967.295. A fórmula é simples: N bits → 0 a (2ᴺ − 1).

🎨
Exemplo prático: cores RGB Cada canal de cor (R, G, B) é um inteiro de 8 bits sem sinal: 0 a 255. É por isso que rgb(255, 128, 0) é laranja — vermelho máximo, verde médio, azul zero. O "255" não é arbitrário: é o máximo de 8 bits.

O problema dos negativos

Para representar negativos, a solução ingênua seria usar 1 bit para o sinal (0 = positivo, 1 = negativo). Mas isso tem um problema: 0 positivo e 0 negativo seriam valores diferentes, e o hardware de soma precisaria de lógica extra para lidar com o sinal.

A solução adotada universalmente é o complemento de dois (two's complement) — uma representação elegante que elimina esses problemas.

Complemento de dois: como funciona

Em complemento de dois com 8 bits: o bit mais significativo (o da esquerda) vale −128 (negativo!), e os outros continuam iguais.

  1. +5
    00000101 = 0 + 4 + 1 = +5
    bit de sinal = 0 (positivo)
  2. −5
    11111011 = −128 + 64 + 32 + 16 + 8 + 2 + 1 = −5
    bit de sinal = 1 (negativo)

Para obter o complemento de dois de um número: inverta todos os bits, depois some 1.

  1. 1
    +5 em binário: 00000101
  2. 2
    Inverta todos os bits: 11111010
  3. 3
    Some 1: 11111011 = −5 ✓

A elegância do complemento de dois: Com essa representação, a mesma ULA que soma positivos soma negativos corretamente. 5 + (−5) = 00000101 + 11111011 = 100000000 — o carry do 9º bit é descartado, sobrando 00000000 = 0. Perfeito, sem nenhuma lógica extra.

Overflow: quando os bits acabam

Overflow acontece quando o resultado de uma operação ultrapassa o range do tipo de dado. O valor "dá a volta" de forma inesperada.

Overflow em unsigned (8 bits)

255 + 1 = 0

11111111 + 00000001 = 100000000. O 9º bit não cabe — é descartado. O resultado é 00000000 = 0. O número "dá a volta" do máximo para zero.

Overflow em signed (8 bits)

127 + 1 = −128

01111111 + 1 = 10000000. O bit de sinal vira 1, tornando o resultado −128. Do maior positivo para o menor negativo em um passo.

🐛
Bug histórico — Ariane 5 (1996): O foguete europeu Ariane 5 explodiu 37 segundos após o lançamento por um bug de overflow: um número de ponto flutuante de 64 bits foi convertido para um inteiro de 16 bits, causando overflow. O sistema de navegação travou e o foguete foi destruído. Custo: ~370 milhões de dólares. Overflow não é trivial.

Tipos inteiros em linguagens de programação

Python

int de tamanho ilimitado

Python aloca mais memória automaticamente conforme o número cresce. Não há overflow em inteiros Python puros — mas isso tem custo de performance. Para precisão de bits, use numpy.int32.

C / C++

Tamanhos fixos

int8_t (−128 a 127), uint8_t (0 a 255), int32_t (±2 bilhões), int64_t (±9 quintilhões). Overflow é silencioso e causa bugs difíceis de detectar.

JavaScript

Number (64-bit float)

JS usa ponto flutuante de 64 bits para todos os números. Inteiros são seguros até 2⁵³. Acima disso, use BigInt. É por isso que 0.1 + 0.2 !== 0.3 em JavaScript.

Rust

Overflow é erro em debug

Rust detecta overflow em modo debug e entra em panic. Em release, comporta-se como C (wrapping). Você pode escolher explicitamente: wrapping_add, saturating_add, checked_add.

Termos-chave desta aula

Unsigned integer Inteiro sem sinal. Representa apenas valores ≥ 0. Um byte unsigned vai de 0 a 255.
Signed integer Inteiro com sinal. Usa complemento de dois. Um byte signed vai de −128 a +127.
Complemento de dois Representação de negativos: inverta todos os bits e some 1. Permite que a mesma ULA opere com positivos e negativos sem circuitos extras.
Bit de sinal O bit mais significativo em complemento de dois. 0 = positivo, 1 = negativo.
Integer overflow Quando um cálculo produz um resultado fora do range do tipo de dado. O valor "dá a volta" de forma inesperada. Causa bugs sérios em linguagens de baixo nível.
