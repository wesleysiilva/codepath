Fase 1 · Representação de Dados

Compressão:
fazer mais com menos bits

Uma música WAV de 3 minutos ocupa ~30 MB. O mesmo áudio em MP3 ocupa ~3 MB. Um vídeo bruto de 4K ocuparia terabytes por hora — o streaming só funciona porque comprimimos. Entenda como reduzir dados sem (ou com pouca) perda de qualidade.

⏱ ~12 min
📶 Iniciante
Módulo 4 de 5

Dois tipos de compressão

Lossless (sem perdas)

Comprime e restaura perfeitamente

O arquivo descomprimido é bit a bit idêntico ao original. Nenhuma informação é perdida. Funciona encontrando e eliminando redundâncias nos dados. Ex: ZIP, PNG, FLAC, zstd.

Lossy (com perdas)

Descarta o imperceptível

Remove informações que o olho ou ouvido humano dificilmente percebe. Irreversível — não é possível recuperar o original. Compressões muito maiores. Ex: JPEG, MP3, AAC, H.264, WebP.

📊
Quando usar cada um: Use lossless quando a precisão importa: código-fonte (ZIP/gzip), imagens que serão editadas (PNG), áudio de produção (FLAC), dados científicos, backups. Use lossy para distribuição final ao usuário: fotos publicadas (JPEG), streaming de música (MP3/AAC), vídeos na web (H.264/H.265).

Como funciona a compressão lossless

A ideia central é explorar padrões e redundâncias nos dados. Dois métodos clássicos:

Run-Length Encoding (RLE) — comprimir sequências repetidas:

Original AAAAAABBBBCCDDDDDDDDDD (22 bytes)
Comprimido 6A 4B 2C 10D (8 bytes) — 64% menor

Huffman Coding — caracteres frequentes recebem códigos menores. Em vez de 8 bits para cada letra, 'e' (mais comum) pode usar 3 bits e 'z' (raro) usa 9 bits. É o que o DEFLATE (usado pelo ZIP e PNG) faz internamente.

📦
Limites da compressão lossless: Dados sem padrão — como ruído aleatório ou dados já comprimidos — não se comprimem mais. Tentar compactar um arquivo JPEG com ZIP pode até aumentar levemente o tamanho. O teorema de Shannon define o limite teórico de compressão: a entropia da informação.

Como funciona a compressão lossy

Compressão lossy aproveita as limitações da percepção humana. O JPEG, por exemplo, usa que o olho humano percebe variações de brilho melhor que variações de cor — e descarta mais informação de cor que de brilho.

JPEG

DCT + quantização

A imagem é dividida em blocos de 8×8 pixels. Cada bloco é transformado por DCT (coseno) em coeficientes de frequência. Frequências altas (detalhes finos) são descartadas com base no nível de qualidade. O famoso "efeito JPEG" visível em baixa qualidade é esse descarte excessivo.

MP3 / AAC

Mascaramento psicoacústico

Sons muito altos "mascaram" sons baixos próximos — o ouvido não percebe os mais baixos. MP3 identifica esses sons mascarados e os descarta. Também remove frequências acima de ~16 kHz (pouco audíveis para adultos). Resultado: 10× menor que WAV com qualidade subjetiva parecida.

H.264 / H.265 (vídeo)

Inter-frame prediction

Em vez de armazenar cada frame completo, o codec armazena apenas as diferenças entre frames consecutivos. Um fundo estático só é guardado uma vez. O personagem movendo na frente é uma "diferença" pequena. H.265 é 40% mais eficiente que H.264 na mesma qualidade.

Comparação de tamanhos

Uma música de 3 minutos em diferentes formatos:

WAV (sem compressão)
~30 MB
FLAC (lossless)
~15 MB
MP3 320kbps
~7 MB
MP3 128kbps
~3 MB

Termos-chave desta aula

Compressão lossless Compressão sem perdas de dados. O arquivo descomprimido é idêntico ao original. Ex: ZIP, PNG, FLAC.
Compressão lossy Compressão com descarte irreversível de dados imperceptíveis. Arquivos muito menores. Ex: JPEG, MP3, H.264.
Run-Length Encoding (RLE) Técnica lossless que substitui sequências repetidas por contador + valor. Ex: AAAAA → 5A.
Huffman Coding Técnica lossless que atribui códigos binários menores a símbolos mais frequentes. Base do ZIP e PNG.
Artefato de compressão Distorção visual ou auditiva causada por compressão lossy excessiva. O efeito de blocos no JPEG em baixa qualidade é o exemplo mais visível.
Bitrate Taxa de bits por segundo em áudio/vídeo (kbps, Mbps). Maior bitrate = mais qualidade + arquivo maior. MP3 128kbps vs. 320kbps é um trade-off qualidade/tamanho.
