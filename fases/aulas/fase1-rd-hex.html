Hexadecimal:
o atalho do binário

Binário é preciso, mas impraticável de ler. Hexadecimal resolve isso: cada dígito hex representa exatamente 4 bits. Cores no CSS, endereços de memória, valores de hash — tudo usa hex porque é muito mais compacto que binário.

⏱ ~12 min
📶 Iniciante
Módulo 4 de 5

Base 16: os 16 dígitos

O hexadecimal usa base 16. Como só temos 10 algarismos (0–9), usamos as letras A–F para os valores de 10 a 15. Um único dígito hex vai de 0 a 15 — o equivalente a 4 bits.

Por que hex é tão prático?

A beleza do hexadecimal é que 1 dígito hex = 4 bits exatos. Isso torna a conversão binário ↔ hex trivial: agrupe os bits em grupos de 4.

  1. ex.
    Binário: 1010 1111 0011 1100
    Grupo a grupo: A + F + 3 + C = 0xAF3C

Sem hex, um endereço de memória de 64 bits seria 64 dígitos binários. Em hex, são apenas 16 dígitos. Isso explica por que todo endereço que você vê em debuggers e dumps de memória está em hex.

🔖
Prefixos de notação: Para distinguir hex de decimal, usa-se o prefixo 0x em código (0xFF, 0x1A3C) ou o sufixo h em assembly (FFh). Em CSS, usa-se o símbolo # (#FF5733). Todos representam hexadecimal.

Cores em hexadecimal (CSS)

Uma cor no CSS como #FF5733 é formada por três bytes em hex — um para cada canal de cor:

#FF5733
R=255 G=87 B=51
#9B8FFF
R=155 G=143 B=255
#5BC4A0
R=91 G=196 B=160
#F0A050
R=240 G=160 B=80

FF em hex = 255 em decimal = máximo do canal. 00 = 0 = desligado. #FF0000 é vermelho puro. #FFFFFF é branco (todos os canais no máximo). #000000 é preto. Agora você lê cores hex diretamente.

Hex no dia a dia do desenvolvedor

Endereços de memória

0x00007fff5fbff000

Debuggers, mensagens de erro de segmentation fault e dumps de memória mostram endereços em hex. Saber ler 0x7F = 127 transforma símbolos em informação.

Hashes criptográficos

SHA-256 em hex

Um hash SHA-256 tem 256 bits = 32 bytes = 64 dígitos hex. É por isso que hashes sempre parecem longas strings de letras e números — elas são números em hex.

Código em Python

0xFF, 0b1010, 0o17

Python aceita literais em hex (0xFF), binário (0b1010) e octal (0o17). hex(255) retorna '0xff'. Útil para trabalhar com bits, cores e protocolos de rede.

Arquivos binários

Hex editors

Para inspecionar arquivos binários (imagens, executáveis, bancos de dados), usa-se um hex editor que mostra os bytes como pares de dígitos hex. Saber ler hex é essencial para debugging de baixo nível.

Termos-chave desta aula

Hexadecimal (base 16) Sistema numérico com 16 símbolos: 0–9 e A–F. Cada dígito representa 4 bits exatos, tornando a conversão com binário trivial.
Prefixo 0x Notação em código para indicar que o valor é hexadecimal. Ex: 0xFF = 255 em decimal.
Nibble Grupo de 4 bits. Um byte tem 2 nibbles. Cada nibble corresponde exatamente a um dígito hexadecimal.
Cor hexadecimal Representação de cores em CSS como 3 bytes hex (RRGGBB). #FF5733 = R:255, G:87, B:51.
