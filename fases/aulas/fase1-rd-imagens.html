Fase 1 · Representação de Dados

Imagens e pixels:
uma grade de números

Uma foto não é "uma imagem" para o computador — é uma grade de milhões de números. Cada pixel é três bytes: R, G, B. Entender isso explica resolução, tamanho de arquivo, formatos PNG vs JPEG e como o processamento de imagens funciona.

⏱ ~12 min
📶 Iniciante
Módulo 4 de 5

O pixel e o modelo RGB

O pixel (picture element) é a menor unidade de uma imagem digital. Cada pixel é representado por três valores numéricos, um para cada canal de cor primária:

R
0 a 255
8 bits
G
0 a 255
8 bits
B
0 a 255
8 bits

Um pixel RGB ocupa 3 bytes (24 bits). Com 256 × 256 × 256 = mais de 16 milhões de combinações possíveis, esses três bytes representam qualquer cor visível pelo olho humano. Imagens com canal alpha (transparência) usam RGBA: 4 bytes por pixel.

Cada quadrado acima é 1 pixel — 3 bytes de dados.

Resolução e tamanho de arquivo

Resolução é o número de pixels — geralmente expresso como largura × altura. Uma imagem "Full HD" tem 1920 × 1080 = 2.073.600 pixels.

  1. ex.
    Foto de smartphone de 12 megapixels: 4032 × 3024 = 12.192.768 pixels
    Sem compressão (RGB puro): 12.192.768 × 3 bytes = ~36,6 MB
  2. ex.
    Wallpaper 4K (3840 × 2160) PNG sem compressão:
    3840 × 2160 × 3 bytes = ~23,7 MB — por isso wallpapers 4K são grandes
📐
DPI vs. resolução: DPI (dots per inch) mede a densidade de pixels na impressão física. Uma imagem de 1000×1000 a 300 DPI mede ~8,5×8,5 cm impressa. A mesma imagem a 72 DPI (tela) ficaria maior. DPI importa para impressão, não para tela.

Formatos de imagem

PNG

Compressão sem perdas

Cada pixel é preservado integralmente — a imagem descomprimida é idêntica ao original. Suporta transparência (canal alpha). Ideal para logos, ícones, capturas de tela e imagens com texto. Arquivos maiores que JPEG.

JPEG

Compressão com perdas

Descarta informações imperceptíveis ao olho humano para reduzir o arquivo. Uma foto de 36 MB em PNG bruto pode virar 3 MB em JPEG com qualidade 80%. Não suporta transparência. Cada vez que você salva, perde mais qualidade.

WebP

O melhor dos dois

Formato moderno do Google. Oferece compressão com e sem perdas, suporta transparência e é ~25–35% menor que JPEG na mesma qualidade. Suportado por todos os browsers modernos. Use em projetos web.

SVG

Vetorial — não é grade de pixels

Descreve formas matemáticas (círculos, curvas, texto) em XML. Escala para qualquer tamanho sem perder qualidade. Arquivos pequenos para logos simples. Não serve para fotos — serve para ícones, gráficos, ilustrações.

Por que isso importa para você, programador?

Processamento de imagens

Pixels são arrays

Com Pillow (Python), uma imagem é uma grade de números. Filtros, brilho, contraste, detecção de bordas — tudo é operação matemática nos arrays de pixels. Saber que cada pixel é RGB torna a lógica natural.

Machine Learning

Imagens como entrada de IA

Redes neurais não "veem" imagens — recebem arrays de números. Uma foto 224×224 RGB entra como tensor de forma (224, 224, 3). Entender pixels torna a preparação de dados para IA muito mais clara.

Web performance

Escolha o formato certo

Fotos → JPEG ou WebP. Ícones e logos → SVG ou PNG. Screenshots com texto → PNG. Animações simples → SVG ou CSS. Usar PNG para fotos ou JPEG para logos são erros comuns que inflam o tamanho das páginas.

Termos-chave desta aula

Pixel Picture element. A menor unidade de uma imagem digital. Em RGB, cada pixel é representado por 3 bytes (R, G, B).
RGB Modelo de cor aditiva com três canais: Red, Green, Blue. Cada canal é um valor de 0 a 255. Combinados, representam mais de 16 milhões de cores.
Resolução Número total de pixels de uma imagem, geralmente expresso como largura × altura (ex: 1920×1080). Determina o tamanho bruto de arquivo.
Canal alpha Quarto canal de cor (A) que representa transparência. RGBA usa 4 bytes por pixel. Suportado por PNG e WebP, não por JPEG.
PNG vs. JPEG PNG = compressão sem perdas (preserva todos os pixels, arquivo maior). JPEG = compressão com perdas (descarta dados imperceptíveis, arquivo menor, sem transparência).
