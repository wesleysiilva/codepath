Texto: ASCII e Unicode:
como letras viram números

O computador não "conhece" letras — só números. Para representar texto, criaram-se tabelas que mapeiam números a caracteres. O ASCII funcionou por décadas, mas não cabia o mundo. O Unicode resolveu — e o UTF-8 é hoje o encoding mais usado na internet.

⏱ ~15 min
📶 Iniciante
Módulo 4 de 5

ASCII: a tabela original

O ASCII (American Standard Code for Information Interchange, 1963) mapeou 128 caracteres para números de 0 a 127. Com 7 bits era possível representar: letras maiúsculas e minúsculas do inglês, dígitos, pontuação e caracteres de controle.

Truque do ASCII: Maiúsculas e minúsculas diferem apenas pelo bit 5. 'A' = 65 = 01000001, 'a' = 97 = 01100001. Para converter de maiúscula para minúscula: some 32 (ou ative o bit 5). É por isso que ord('a') - ord('A') == 32 em Python.

O problema do ASCII: só inglês

128 caracteres não cabem "ã", "é", "ç", "漢字", "مرحبا", "🎉". O ASCII foi estendido para 256 caracteres (Latin-1, ISO-8859-1) — o que coube os acentos do português, mas deixou de fora toda a Ásia, o Árabe, o Cirílico e os emojis.

Cada país criou seu próprio encoding — ISO-8859-1 para Europa Ocidental, Shift-JIS para o japonês, GB2312 para o chinês. Um arquivo criado no Japão aberto em um computador europeu exibia caracteres incompreensíveis.

Unicode: uma tabela para o mundo inteiro

O Unicode (1991) definiu um espaço com mais de 1 milhão de posições — code points — para representar todos os caracteres de todos os sistemas de escrita humanos conhecidos, mais símbolos matemáticos, emojis, e caracteres históricos.

Code point

U+XXXX

Cada caractere Unicode tem um número único chamado code point. 'A' = U+0041, 'ç' = U+00E7, '漢' = U+6F22, '😀' = U+1F600. O Unicode define o número — não como armazená-lo em bytes.

Encoding

Como representar em bytes

Unicode define os números; o encoding define como salvar esses números em bytes no disco ou na rede. Os encodings principais são: UTF-8, UTF-16 e UTF-32.

UTF-8: o encoding dominante

O UTF-8 é o encoding mais usado na internet (usada em mais de 98% das páginas web). Sua elegância: caracteres ASCII usam 1 byte (compatível com ASCII!), e caracteres mais raros usam 2, 3 ou 4 bytes conforme necessário.

Texto: "Olá"
Code points: U+004F U+006C U+00E1
UTF-8 bytes:
4F 6C C3 A1
Verde = 1 byte (ASCII, U+0000–U+007F) · Roxo = 2 bytes (U+0080–U+07FF)
🔥
Mojibake — o fantasma do encoding errado: Aquele "Ã©" no lugar de "é" que você já viu? Chama-se mojibake (文字化け, japonês). Acontece quando um texto em UTF-8 é lido como Latin-1. O byte C3 A9 (= "é" em UTF-8) é interpretado como dois caracteres Latin-1: "Ã" (0xC3) e "©" (0xA9). Sempre especifique o encoding!

Por que isso importa para você, programador?

Python 3

str vs. bytes

Em Python 3, str é Unicode. bytes é sequência de bytes brutos. Para converter: "olá".encode("utf-8") → bytes; b"\xc3\xa1".decode("utf-8") → str. Confundir os dois é erro comum.

Arquivos

Sempre especifique encoding

open("arquivo.txt", encoding="utf-8"). Sem especificar, Python usa o encoding do SO — que varia. No Windows pode ser CP1252, no Linux UTF-8. Código que funciona em Linux pode quebrar no Windows.

len() e caracteres

Cuidado com multibyte

len("é") = 1 em Python (conta caracteres). Mas len("é".encode("utf-8")) = 2 (conta bytes). Para sistemas que medem tamanho em bytes (HTTP, banco de dados), use bytes, não len().

Emojis e len()

Emojis são 1 code point

len("😀") = 1 em Python. Mas em bytes UTF-8 ocupa 4 bytes. Alguns emojis compostos (família 👨‍👩‍👧) têm múltiplos code points ligados com ZWJ — len() retorna um número surpreendente.

Termos-chave desta aula

ASCII Tabela de 128 caracteres (7 bits) criada em 1963. Define o mapeamento número→caractere para o inglês. É um subconjunto do Unicode.
Unicode Padrão que define um code point único (U+XXXX) para cada caractere de todos os sistemas de escrita humanos. Mais de 140.000 caracteres definidos.
UTF-8 Encoding de comprimento variável para Unicode. ASCII usa 1 byte; caracteres até U+07FF usam 2; até U+FFFF usam 3; restante usa 4. Compatível com ASCII, dominante na web.
Code point Número único que identifica um caractere no Unicode. Escrito como U+XXXX. Ex: 'A' = U+0041, '😀' = U+1F600.
Encoding Regra que define como representar code points Unicode como sequência de bytes. UTF-8, UTF-16 e Latin-1 são encodings diferentes.
Mojibake Caracteres corrompidos exibidos quando um texto é lido com o encoding errado. Ex: "é" vira "Ã©" ao ler UTF-8 como Latin-1.
