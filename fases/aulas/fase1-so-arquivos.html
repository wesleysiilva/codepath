Sistema de arquivos:
a biblioteca do SO

O sistema de arquivos organiza tudo que existe no disco — programas, configurações, documentos, código-fonte. Entender sua hierarquia, caminhos e permissões é essencial para qualquer desenvolvedor, especialmente ao trabalhar com o terminal.

⏱ ~12 min
📶 Iniciante
Módulo 3 de 5

A hierarquia de diretórios

Todo sistema de arquivos moderno é organizado em uma árvore hierárquica de diretórios (pastas). Existe sempre um ponto de partida — a raiz — a partir do qual tudo se ramifica.

Linux / macOS

Raiz: /

Um único ponto de partida para tudo. Discos externos são "montados" em subdiretórios como /mnt ou /media. Tudo está sob /.

Windows

Raiz: letras de drive (C:\)

Cada volume tem sua própria raiz. O drive principal costuma ser C:\; pen drives aparecem como D:\, E:\, etc. Múltiplas árvores independentes.

Estrutura típica de um sistema Linux:

/ (raiz do sistema)
├── bin/ programas essenciais (ls, cp, bash)
├── etc/ arquivos de configuração do sistema
├── home/ pastas pessoais dos usuários
│ └── wesley/ sua pasta home (~)
│ ├── Documents/
│ └── projetos/
├── tmp/ arquivos temporários (apagados no reboot)
├── usr/ programas instalados pelo usuário
└── var/ logs, dados que mudam constantemente

Caminhos absolutos e relativos

Para apontar para um arquivo, você usa um caminho. Existem dois tipos, e entender a diferença é crucial para programação:

Caminho absoluto

A partir da raiz

Começa sempre da raiz (/ no Unix, C:\ no Windows). Funciona de qualquer lugar que você esteja.

Linux: /home/wesley/projetos/app.py
Windows: C:\Users\Wesley\projetos\app.py

Caminho relativo

A partir do diretório atual

Relativo ao "diretório de trabalho atual" (working directory). Mais curto, mas muda de significado conforme onde você está.

projetos/app.py → pasta projetos dentro de onde estou
../outro/app.py → .. sobe um nível

📂
Atalhos de caminho universais:
. (ponto) = diretório atual
.. (dois pontos) = diretório pai (um nível acima)
~ (til, Unix) = pasta home do usuário atual (/home/wesley)
/ no início = caminho absoluto a partir da raiz

Permissões: quem pode fazer o quê

No Unix/Linux, todo arquivo e diretório tem permissões que definem quem pode ler, escrever ou executar. Isso é fundamental para segurança — e explica muitos erros que você encontrará ao desenvolver.

r
Read (leitura)
Pode ler o conteúdo do arquivo ou listar o conteúdo do diretório.
w
Write (escrita)
Pode modificar o arquivo ou criar/apagar arquivos no diretório.
x
Execute (execução)
Pode executar o arquivo como programa, ou entrar no diretório com cd.

Essas permissões são definidas para três grupos: o dono do arquivo, o grupo ao qual pertence, e outros (todos os demais). O comando ls -l mostra isso como uma string de 9 caracteres:

-rwxr-xr-- wesley staff 4096 app.py

rwx = dono pode ler, escrever e executar
r-x = grupo pode ler e executar (não escrever)
r-- = outros só podem ler
🔢
Permissões em octal: Permissões também são representadas como números: r=4, w=2, x=1. Some os valores: rwx = 7, rw- = 6, r-- = 4. chmod 755 script.py define rwxr-xr-x — dono tem tudo, outros só leitura/execução. Você verá isso frequentemente ao configurar servidores.

Formatos de sistema de arquivos

Windows

NTFS

New Technology File System. Suporta arquivos grandes, permissões, journaling (recuperação após falha) e links simbólicos. Padrão do Windows desde o NT 4.0.

Linux

ext4

Fourth Extended Filesystem. O mais comum no Linux. Robusto, com journaling, suporta volumes de até 1 exabyte. Alternativas modernas: btrfs, XFS para casos específicos.

macOS

APFS

Apple File System (2017). Otimizado para SSDs, suporta snapshots instantâneos (Time Machine rápido) e criptografia nativa. Substituiu o HFS+ nos Macs modernos.

Universal

FAT32 / exFAT

Usados em pen drives e cartões SD por compatibilidade total entre OS. FAT32 tem limite de 4 GB por arquivo; exFAT remove esse limite. Sem permissões Unix.

Termos-chave desta aula

Sistema de arquivos Estrutura que o SO usa para organizar, armazenar e recuperar dados no disco. Define hierarquia, nomes, permissões e metadados.
Caminho absoluto Endereço completo de um arquivo a partir da raiz do sistema. Ex: /home/user/projeto/app.py. Funciona de qualquer lugar.
Caminho relativo Endereço de um arquivo em relação ao diretório de trabalho atual. Ex: projeto/app.py. Muda de significado conforme onde você está.
Permissões (rwx) Controle de acesso a arquivos: r (leitura), w (escrita), x (execução). Definidas separadamente para dono, grupo e outros.
chmod Comando Unix para mudar permissões de arquivo. chmod 755 arquivo define rwxr-xr-x.
Journaling Mecanismo do sistema de arquivos que registra operações antes de executá-las, permitindo recuperação em caso de falha de energia ou crash.
