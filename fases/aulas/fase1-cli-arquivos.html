Fase 1 · Linha de Comando

Manipulação de arquivos:
cat, cp, mv e pipes

Ler arquivos, copiá-los, movê-los, redirecionar saídas — e encadear comandos com pipes para criar fluxos poderosos. Esta é a base para automatizar tarefas no terminal.

⏱ ~15 min
📶 Iniciante
Módulo 5 de 5

Ler, copiar e mover arquivos

bash — operações com arquivos
# Criar arquivo com texto
$ echo "Olá, mundo" > ola.txt

# Ler conteúdo de arquivo
$ cat ola.txt
Olá, mundo

# Copiar arquivo
$ cp ola.txt copia.txt

# Mover / renomear arquivo
$ mv copia.txt backup/ola-backup.txt

# Ver as primeiras/últimas linhas de um arquivo grande
$ head -5 log.txt # primeiras 5 linhas
$ tail -20 log.txt # últimas 20 linhas
$ tail -f log.txt # acompanha em tempo real
cat

Concatenate

Lê e exibe o conteúdo de arquivos. cat a.txt b.txt concatena e exibe os dois. Ideal para arquivos pequenos — para grandes, use less (navegável com setas).

cp

Copy

cp origem destino. Para copiar pasta inteira: cp -r pasta/ destino/. O original permanece intacto.

mv

Move (e renomear)

mv arquivo.txt pasta/ move o arquivo. mv nome-antigo.txt nome-novo.txt renomeia. O mv é como cp + rm em um passo.

grep

Buscar padrões em texto

grep "erro" log.txt imprime todas as linhas que contêm "erro". grep -r "def main" src/ busca recursivamente em uma pasta. Essencial para analisar logs.

Redirecionamento: >, >> e <

Por padrão, a saída de um comando vai para a tela (stdout). Com redirecionamento, você manda essa saída para um arquivo:

bash — redirecionamento
# > sobrescreve o arquivo
$ echo "Linha 1" > arquivo.txt
$ echo "Linha 2" > arquivo.txt
$ cat arquivo.txt
Linha 2 (Linha 1 foi sobrescrita!)

# >> acumula (append)
$ echo "Linha 1" >> log.txt
$ echo "Linha 2" >> log.txt
$ cat log.txt
Linha 1
Linha 2

# Salvar a lista de arquivos em um arquivo
$ ls -la > lista.txt
⚠️
Cuidado com o >: comando > arquivo.txt sobrescreve o arquivo sem aviso. Se o arquivo continha dados importantes, eles se foram. Use >> quando quiser acumular (como logs). Use > quando quer criar/substituir limpo.

Pipes (|): encadeando comandos

O pipe (|) conecta a saída de um comando à entrada do próximo. É a filosofia Unix em ação: ferramentas pequenas, cada uma fazendo uma coisa, combinadas para resolver problemas complexos.

ls -la
lista arquivos
|
grep ".py"
filtra .py
|
sort
ordena
|
wc -l
conta linhas
bash — pipes em ação
# Quantos arquivos .py estão nesta pasta?
$ ls *.py | wc -l
14

# Buscar "erro" no log e mostrar só as últimas 10 ocorrências
$ grep "erro" app.log | tail -10

# Processos que consomem mais de 100MB de RAM
$ ps aux | grep -v grep | awk '$6 > 100000'

# Contar palavras únicas em um arquivo de texto
$ cat texto.txt | tr ' ' '\n' | sort | uniq -c | sort -rn | head -10

Termos-chave desta aula

cat Exibe o conteúdo de um arquivo. cat a.txt b.txt concatena os dois.
cp / mv cp copia arquivo (original fica intacto). mv move ou renomeia (original desaparece da origem).
grep Filtra linhas de texto por padrão. grep "palavra" arquivo. Com -r busca recursivamente em diretórios.
Redirecionamento (> e >>) > envia saída para arquivo (sobrescreve). >> acumula ao final do arquivo (append).
Pipe (|) Conecta a saída de um comando à entrada do próximo. Permite encadear ferramentas simples para resolver problemas complexos.
tail -f Acompanha um arquivo em tempo real, exibindo novas linhas conforme são adicionadas. Essencial para monitorar logs de aplicações.
