Fase 1 · Arquitetura de Von Neumann

Registradores:
a memória dentro da CPU

Registradores são pequenos espaços de armazenamento dentro do próprio chip da CPU. São poucos, minúsculos e incrivelmente rápidos — centenas de vezes mais rápidos que a RAM. Qualquer dado que a CPU processa passa por eles.

⏱ ~12 min
📶 Iniciante
Módulo 1 de 5

O que são registradores?

Imagine que a CPU é uma calculadora avançada. Para somar dois números, ela precisa tê-los "em mãos" — não na gaveta (RAM), não no armário (SSD). Registradores são o equivalente a segurar valores nas mãos: acesso imediato, sem espera, sem deslocamento.

Eles ficam fisicamente dentro do chip da CPU, fabricados com os transistores mais rápidos disponíveis. Uma CPU moderna x86-64 tem dezenas de registradores visíveis ao programador — mais centenas de registradores internos usados pelo próprio processador.

Velocidade em perspectiva: Acessar um registrador leva 1 ciclo de clock. Acessar a RAM leva ~100 ciclos. Se você esperasse a RAM toda vez, a CPU ficaria 99% do tempo ociosa. Por isso a CPU trabalha sempre com registradores — e busca dados da RAM para dentro deles antes de operar.

Registradores de uso geral (x86-64)

Na arquitetura x86-64 (usada nos PCs), há 16 registradores de uso geral, cada um com 64 bits. Historicamente, alguns tinham papéis específicos, mas hoje são amplamente intercambiáveis. Os mais conhecidos:

RAX
Accumulator
Usado para resultados de operações e retorno de funções
RBX
Base
Registrador de uso geral, preservado entre chamadas
RCX
Counter
Contador em loops e operações de repetição
RDX
Data
Dados em operações de I/O e multiplicação/divisão
RSP
Stack Pointer
Aponta para o topo da pilha de execução
RBP
Base Pointer
Referência à base do frame atual da pilha
RSI
Source Index
Ponteiro de origem em operações de cópia
RDI
Dest. Index
Ponteiro de destino em operações de cópia
📏
64, 32, 16 e 8 bits no mesmo registrador: RAX é de 64 bits. Mas você pode acessar os 32 bits inferiores como EAX, os 16 bits inferiores como AX, e até os 8 bits inferiores como AL. Isso é herança da evolução do x86 desde os anos 70 — mantida para compatibilidade.

Registradores especiais

Além dos registradores de uso geral, existem registradores com funções fixas que a CPU usa para controlar a execução:

RIP
Instruction Pointer
O Program Counter do x86-64. Aponta para a próxima instrução a ser executada.
RFLAGS
Flags Register
Bits de status: ZF (zero), CF (carry), SF (sinal), OF (overflow). Controlam saltos condicionais.
IR
Instruction Register
Guarda a instrução atual sendo decodificada pela Unidade de Controle.
MAR
Mem. Address Register
Armazena o endereço de memória a ser acessado antes de colocá-lo no barramento.
MDR
Mem. Data Register
Armazena o dado lido ou a ser escrito na memória durante uma operação de acesso.
XMM0–15
SIMD Registers
128 bits cada. Operam com múltiplos números ao mesmo tempo — usados em gráficos, áudio e IA.

Registradores e variáveis: a ligação invisível

Quando você declara uma variável em Python — x = 42 — o interpretador ou o compilador precisa decidir onde guardar esse valor durante a execução. A resposta quase sempre envolve registradores.

  1. 1
    Declaração: o valor 42 está no código-fonte como constante.
  2. 2
    Carregamento: a instrução MOV carrega 42 em um registrador (ex: RAX ← 42).
  3. 3
    Uso: qualquer operação com x usa diretamente o registrador — sem ir à RAM.
  4. 4
    Derrame (spill): se ficarem sem registradores disponíveis, o compilador "derrama" um valor de volta para a RAM temporariamente.
🏎️
Register allocation — a arte do compilador: Compiladores como o do C++ passam uma fase inteira chamada register allocation: decidir quais variáveis ficam em registradores e quais vão para a RAM. Isso é um dos fatores que faz gcc -O2 produzir código 2–5× mais rápido — o otimizador mantém valores quentes nos registradores em vez de ir e voltar à memória.

Por que isso importa para você, programador?

Pilha de execução

RSP e o stack frame

Toda vez que você chama uma função, a CPU usa RSP para criar um "stack frame" — um espaço na memória para os parâmetros e variáveis locais daquela chamada. Entender isso explica recursão infinita (stack overflow) e por que funções têm escopo próprio.

Debugging de baixo nível

Ver registradores no debugger

Ferramentas como GDB ou o debugger do VS Code podem mostrar o estado dos registradores durante a execução. Saber que RAX contém o valor de retorno de uma função ajuda a entender crashes e comportamentos inesperados.

Performance

Variáveis locais vs. globais

Variáveis locais têm mais chance de ficar em registradores (mais rápido). Variáveis globais e acessos a atributos de objetos costumam exigir mais acessos à memória. Não é motivo para refatorar prematuramente, mas explica diferenças de performance.

Assembly e segurança

A base das vulnerabilidades

Buffer overflows, ROP chains e muitos exploits clássicos manipulam diretamente registradores como RSP e RIP para redirecionar a execução do programa. Entender registradores é o primeiro passo para entender segurança de baixo nível.

Termos-chave desta aula

Registrador Pequeno espaço de armazenamento dentro da CPU, acessível em 1 ciclo. Toda operação aritmética ou lógica passa por registradores.
RAX / RBX / RCX... Registradores de uso geral de 64 bits na arquitetura x86-64. "R" = 64 bits, "E" = 32 bits inferiores, sem prefixo = 16 bits.
RIP (Instruction Pointer) O Program Counter do x86-64. Aponta para o endereço da próxima instrução a ser buscada no ciclo FDE.
RFLAGS Registrador de status. Contém bits (ZF, CF, SF, OF) que refletem o resultado da última operação e controlam os saltos condicionais.
RSP (Stack Pointer) Aponta para o topo da pilha de execução. Manipulado automaticamente ao chamar e retornar de funções.
Register allocation Fase de otimização do compilador que decide quais variáveis ficam em registradores para maximizar velocidade.
Spill Quando o compilador não tem registradores suficientes e precisa mover um valor temporariamente para a RAM.
