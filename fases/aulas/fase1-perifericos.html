Periféricos I/O:
a ponte entre você e o computador

Teclado, mouse, monitor, câmera, caixas de som — todos são periféricos. Eles são a interface entre o mundo humano e o mundo digital. Entenda o que são dispositivos de entrada e saída, como eles se comunicam com o computador e o que acontece a cada tecla que você pressiona.

⏱ ~15 min
📶 Iniciante
Módulo 1 de 5

O que são periféricos?

Periféricos são todos os dispositivos externos ao núcleo do computador (CPU + RAM + placa-mãe) que permitem a comunicação com o mundo externo. O nome vem de "periferia" — estão na borda do sistema, não no centro.

I/O significa Input/Output — Entrada e Saída. Todo periférico faz uma dessas duas coisas (ou as duas):

Input (Entrada)

Do mundo para o computador

Capturam informações do mundo real e as enviam para o computador processar. Teclado, mouse, microfone, câmera, scanner, joystick, touchpad.

Output (Saída)

Do computador para o mundo

Recebem dados processados e os apresentam ao usuário ou ao ambiente. Monitor, caixas de som, fones de ouvido, impressora, projetor.

I/O (Entrada e Saída)

Os dois sentidos

Comunicam em ambas as direções. Pen drive (lê e escreve), touchscreen (toca = entrada, exibe = saída), placa de rede (envia e recebe dados).

Armazenamento

Tecnicamente também é I/O

HDs e SSDs são dispositivos de armazenamento, mas também se enquadram como I/O — o computador lê e escreve neles constantemente.

Dispositivos de entrada em detalhe

Vamos entender o que acontece por dentro quando você usa os periféricos mais comuns:

  1. ⌨️
    Teclado Cada tecla é um interruptor (switch). Quando você pressiona uma tecla, um circuito fecha, gerando um sinal elétrico. O controlador do teclado converte isso num scan code — um número que identifica qual tecla foi pressionada. Esse código vai para o sistema operacional via USB (ou Bluetooth), que o converte no caractere correspondente considerando o idioma e o layout configurado.
  2. 🖱️
    Mouse óptico Um LED ilumina a superfície abaixo do mouse. Um sensor CMOS captura fotos minúsculas dessa superfície centenas de vezes por segundo e compara os frames para detectar movimento. O controlador calcula a direção e velocidade do movimento e envia as coordenadas delta (quanto moveu em X e Y) para o sistema operacional.
  3. 🎙️
    Microfone Ondas de pressão sonora movem uma membrana. Esse movimento mecânico é convertido em sinal elétrico analógico. O conversor ADC (Analog-to-Digital Converter) digitaliza esse sinal — transforma as variações de tensão em amostras numéricas dezenas de milhares de vezes por segundo (44.100 Hz para qualidade de CD).
  4. 📷
    Câmera / Webcam A lente foca a luz em um sensor CMOS ou CCD. Cada ponto do sensor mede a intensidade da luz vermelha, verde e azul. Esses valores RGB são digitalizados e montados no frame da imagem. Uma webcam 1080p captura 1.920 × 1.080 = ~2 milhões de pixels, 30 vezes por segundo.

Dispositivos de saída em detalhe

  1. 🖥️
    Monitor A GPU calcula a cor de cada pixel e armazena num buffer de frame (framebuffer) na VRAM. O monitor lê esse buffer e acende os subpixels RGB correspondentes. Em 60 Hz, isso acontece 60 vezes por segundo. Em 144 Hz, 144 vezes. Mais Hz = imagem mais fluida, menos rastro em movimento.
  2. 🔊
    Caixas de som / fones O processo inverso do microfone: dados digitais são convertidos em sinal analógico pelo DAC (Digital-to-Analog Converter). Esse sinal elétrico faz vibrar um eletroímã, que move uma membrana, que cria ondas de pressão no ar — o som que você ouve.
  3. 🖨️
    Impressora O computador envia dados da página para a impressora via USB ou rede. A impressora processa esse arquivo (muitas vezes em formato PDF ou PostScript) e usa jatos de tinta ou laser para reproduzir a imagem no papel, ponto a ponto.

A cadeia completa: do teclado à tela

Vamos rastrear o que acontece desde o momento em que você pressiona a tecla "A" no teclado até o momento em que o caractere aparece na tela do editor de código:

  1. 1
    Pressionar a tecla O switch da tecla "A" fecha o circuito. O controlador do teclado detecta e gera o scan code correspondente.
  2. 2
    USB envia o evento O teclado envia o evento via USB para o controlador USB da placa-mãe. O protocolo USB define pacotes de dados com tipo de evento, scan code e estado das teclas modificadoras (Shift, Ctrl...).
  3. 3
    Interrupção de hardware O controlador USB gera uma interrupção — um sinal para a CPU parando o que ela está fazendo e atendendo o evento do teclado. O sistema operacional recebe e processa o scan code.
  4. 4
    Tradução e entrega ao aplicativo O SO converte o scan code em um keycode virtual ("tecla A"), considerando idioma e modificadores. Esse evento é colocado na fila de mensagens do aplicativo com foco (o editor de código).
  5. 5
    O editor processa O editor de código recebe o evento, insere o caractere "a" (ou "A" com Shift) no documento, atualiza o estado interno e marca a tela para ser redesenhada.
  6. 6
    A GPU renderiza O editor pede ao sistema de renderização (via DirectX, Metal ou OpenGL) para desenhar o cursor e o novo caractere. A GPU renderiza os pixels e o monitor exibe o resultado.
⚡
Tudo isso em menos de 16 ms. Em um sistema bem otimizado, desde o toque na tecla até o pixel aparecer na tela demora menos de um frame de 60 Hz (16,6 ms). É por isso que monitores de alta taxa de atualização (144 Hz, 240 Hz) fazem diferença em jogos competitivos — menos tempo entre ação e feedback visual.

Como periféricos se conectam: USB, HDMI e afins

USB

Universal Serial Bus

O conector mais comum para periféricos. USB-A (o retangular de sempre), USB-C (oval, reversível). USB 3.0 transfere até 5 Gbps; USB 3.2 até 20 Gbps; Thunderbolt 4 até 40 Gbps.

HDMI / DisplayPort

Vídeo e áudio

HDMI é o padrão em TVs e monitores de consumo. DisplayPort é preferido em monitores para PC, suporta taxas de atualização maiores. Ambos transmitem vídeo + áudio num único cabo.

Bluetooth

Sem fio de curto alcance

Protocolo de rádio de curto alcance (~10 m). Usado em mouses, teclados, fones e gamepads sem fio. Consome menos energia que Wi-Fi — ideal para dispositivos com bateria.

Placa de rede

I/O para a internet

Também é um periférico de I/O — envia e recebe dados pela rede. Ethernet (cabo RJ45) para conexões cabeadas; Wi-Fi para sem fio. A placa de rede é a porta de entrada da internet no seu computador.

Por que isso importa para você, programador?

Quando você programa, você está criando software que, em algum momento, vai receber entrada do usuário e produzir alguma saída. Entender I/O em nível de hardware ajuda a entender por que certas operações são lentas e como estruturar programas melhores.

input() em Python

Espera do teclado

Quando seu programa chama input(), ele fica parado esperando o usuário digitar. Esse bloqueio vem diretamente do mecanismo de I/O do sistema operacional.

print() em Python

Escrita na saída padrão

print() escreve na stdout — a saída padrão, que normalmente é o terminal. Isso é I/O de saída. Em sistemas embarcados, pode ir para um display físico.

Leitura de arquivos

I/O de disco

Abrir e ler um arquivo é uma operação de I/O. É por isso que leitura de arquivo é mais lenta que ler uma variável na RAM — envolve o controlador do SSD/HD.

I/O assíncrono

Não bloquear enquanto espera

Programas modernos não ficam parados enquanto esperam I/O. Usam técnicas assíncronas (async/await) para continuar processando outras coisas. Você vai aprender isso em fases mais avançadas.

Termos-chave desta aula

Periférico Dispositivo externo ao núcleo (CPU/RAM) que comunica o computador com o mundo externo.
I/O Input/Output — Entrada e Saída. Todo dado que entra ou sai do processamento central.
Scan code Código numérico enviado pelo teclado identificando qual tecla foi pressionada (antes da tradução para caractere).
Interrupção (IRQ) Sinal que um periférico envia para a CPU pedindo atenção imediata. Faz a CPU pausar e atender o evento.
ADC / DAC Analog-to-Digital Converter / Digital-to-Analog Converter. Converte entre sinais analógicos (som, luz) e digitais.
Framebuffer Área na VRAM onde a GPU armazena os pixels do frame atual antes de enviá-los ao monitor.
Taxa de atualização (Hz) Quantas vezes por segundo o monitor exibe um novo frame. 60 Hz = 60 frames/s. Mais Hz = mais fluido.
stdout / stdin Saída padrão e entrada padrão no terminal. São as abstrações de I/O mais básicas que um programa usa.
