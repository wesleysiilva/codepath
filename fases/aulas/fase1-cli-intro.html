Fase 1 · Linha de Comando

Terminal e shell:
falar diretamente com o SO

A interface gráfica é uma camada de conveniência — por baixo, tudo é texto. O terminal é a porta de entrada para o controle real do sistema. Todo desenvolvedor usa o terminal diariamente: instalar pacotes, rodar código, usar Git.

⏱ ~12 min
📶 Iniciante
Módulo 5 de 5

Terminal, shell, prompt — qual a diferença?

Esses três termos são frequentemente confundidos. Eles se referem a coisas relacionadas mas distintas:

Terminal

A janela

O programa que exibe a interface de texto. No macOS: Terminal.app ou iTerm2. No Windows: Windows Terminal. No Linux: GNOME Terminal, Konsole. O terminal hospeda o shell — ele é só a tela.

Shell

O interpretador

O programa que interpreta seus comandos e os envia ao SO. Bash (Linux/Mac antigo), Zsh (macOS padrão desde 2019), Fish (amigável), PowerShell (Windows moderno). O shell é onde a lógica acontece.

Prompt

O convite para digitar

O texto que o shell exibe para indicar que está pronto para receber um comando. Geralmente mostra usuário, máquina e diretório atual. O $ no final indica que você não é root; # indica root.

bash — wesley@ubuntu
# Anatomia do prompt
wesley@ubuntu:~/projetos$ _

# usuário @ máquina : diretório atual $
# ~ significa a pasta home do usuário

Por que usar o terminal?

A interface gráfica é eficiente para tarefas visuais. O terminal é imbatível para:

Velocidade

Um comando vs. 10 cliques

Criar 50 pastas, renomear 200 arquivos, instalar 10 pacotes — o que levaria minutos na GUI leva segundos no terminal. Tarefas repetitivas são automatizáveis com scripts.

Ferramentas de dev

Git, npm, pip, docker...

As ferramentas mais importantes do desenvolvimento moderno não têm GUI — elas vivem no terminal. Git, pip, npm, cargo, docker, kubectl. Saber o terminal é pré-requisito para usá-las.

Servidores

Sem GUI no servidor

Servidores Linux em produção geralmente não têm interface gráfica. Para fazer deploy, configurar nginx, ler logs ou debugar em produção, o terminal é o único caminho.

Automação

Scripts shell

Comandos no terminal podem ser salvos em scripts (.sh, .ps1) e executados automaticamente. Backups, deploy, testes, geração de relatórios — qualquer tarefa repetível pode ser automatizada.

Shells mais comuns

Bash

Bourne Again Shell

O shell mais universal. Padrão no Linux há décadas. Scripts bash rodam em praticamente qualquer servidor. Aprenda bash e você pode trabalhar em qualquer ambiente Unix.

Zsh

Padrão no macOS (2019+)

Compatível com bash, mas com melhorias: autocomplete avançado, sugestões de comando, personalização fácil com Oh My Zsh. Se você usa Mac, já está no Zsh.

PowerShell

Shell moderno do Windows

Muito mais poderoso que o CMD antigo. Trabalha com objetos .NET em vez de texto puro. Disponível também no Linux/Mac. Preferível ao CMD para desenvolvimento no Windows.

Fish

Friendly Interactive Shell

Foco em ser amigável: sugestões automáticas baseadas no histórico, sintaxe colorida, ajuda embutida. Excelente para iniciantes. Não é compatível com scripts bash — avalie bem antes de adotar.

💡
Por onde começar: Se você usa Mac: já tem Zsh, abra o Terminal.
Se usa Windows: instale o WSL (Ubuntu) ou use o Git Bash. O Windows Terminal com PowerShell também funciona para começar.
Se usa Linux: já tem Bash. Abra o terminal do seu desktop environment.

Seu primeiro comando

Abra o terminal e digite — echo "Olá, terminal!". O echo imprime texto na tela. Parabéns — você executou seu primeiro comando.

bash
$ echo "Olá, terminal!"
Olá, terminal!

$ whoami
wesley

$ date
Sat Apr 19 10:30:00 -03 2025

$ python3 --version
Python 3.12.0

Termos-chave desta aula

Terminal Programa que fornece a interface de janela de texto. Hospeda o shell. Ex: Terminal.app, Windows Terminal, iTerm2.
Shell Interpretador de comandos que roda dentro do terminal. Recebe texto, interpreta e envia ao SO. Ex: Bash, Zsh, PowerShell.
Prompt Texto que o shell exibe para indicar que está aguardando entrada. Geralmente mostra usuário, host e diretório atual. $ = usuário comum, # = root.
Bash Bourne Again Shell. Shell mais universal em sistemas Unix/Linux. Padrão em servidores. Scripts bash são portáveis para quase qualquer ambiente.
CLI Command Line Interface. Interface de linha de comando — qualquer programa operado por texto no terminal, em contraste com GUI (interface gráfica).
