Fase 1 · Arquitetura de Von Neumann

ULA: a calculadora
do processador

Toda operação que o computador realiza — somar, comparar, verificar condições — passa pela Unidade Lógica e Aritmética. Entenda como ela funciona, o que são operações lógicas e por que o if do seu código vira uma comparação em hardware.

⏱ ~20 min
📶 Iniciante
Módulo 1 de 5

O que é a ULA?

A ULA (Unidade Lógica e Aritmética — em inglês, ALU: Arithmetic Logic Unit) é o componente da CPU que executa de fato as operações. Enquanto a Unidade de Controle interpreta e coordena, a ULA calcula.

Ela recebe dois operandos (valores de entrada) e um sinal de controle (que operação fazer), executa a operação e produz um resultado — que vai para um registrador — e atualiza flags que indicam propriedades do resultado (foi zero? Houve overflow? O resultado foi negativo?).

🔢
Tudo passa pela ULA. Quando você escreve resultado = a + b em Python, em algum ponto da execução dois valores chegam como operandos na ULA com o sinal de controle "soma", e o resultado sai. A ULA faz isso bilhões de vezes por segundo.

Operações aritméticas

O lado "aritmético" da ULA executa operações matemáticas com números. As fundamentais são:

Adição

A + B

A operação mais fundamental. Em binário: 0101 + 0011 = 1000 (5 + 3 = 8). Subtração é implementada como soma com número negativo (complemento de dois).

Subtração

A − B

Implementada como A + (−B). O número negativo é representado em complemento de dois — um truque binário que permite subtrair usando o mesmo circuito de adição.

Multiplicação / Divisão

A × B / A ÷ B

CPUs modernas têm circuitos dedicados para multiplicação e divisão inteira. Internamente, multiplicação é implementada como somas repetidas e divisão como subtrações repetidas — mas aceleradas por hardware.

Deslocamento de bits

Shift left / right

Mover todos os bits para esquerda ou direita. Deslocar 1 bit à esquerda equivale a multiplicar por 2; à direita, dividir por 2. Muito mais rápido que multiplicação — compiladores usam isso automaticamente.

Operações lógicas

O lado "lógico" da ULA opera bit a bit, usando as regras da lógica booleana. Cada operação compara os bits dos dois operandos posição por posição:

AND
AND — E lógico: resultado é 1 somente se ambos os bits forem 1.
1100 AND 1010 = 1000
Uso: mascarar bits, verificar se um flag está ativo.
OR
OR — OU lógico: resultado é 1 se pelo menos um dos bits for 1.
1100 OR 1010 = 1110
Uso: ativar flags, combinar permissões.
XOR
XOR — OU exclusivo: resultado é 1 se os bits forem diferentes.
1100 XOR 1010 = 0110
Uso: criptografia, detectar diferenças, trocar valores sem variável auxiliar.
NOT
NOT — inversão: inverte todos os bits (0→1, 1→0).
NOT 1100 = 0011
Uso: negar condições, criar complemento de dois para números negativos.

Comparações e o registrador de flags

A ULA também realiza comparações. Quando você escreve if x > 10, em algum momento a ULA compara dois valores e o resultado não vai para um registrador de dados — vai para o registrador de flags (também chamado de registrador de status).

Zero Flag (ZF)

O resultado foi zero?

Ativado quando o resultado de uma operação é zero. Usado para verificar igualdade: a - b == 0 significa a == b.

Carry Flag (CF)

Houve "vai um"?

Ativado quando uma soma gera um bit de carry (vai um) além da largura do registrador. Indica overflow em números sem sinal.

Sign Flag (SF)

O resultado é negativo?

Ativado quando o bit mais significativo do resultado é 1 — que em complemento de dois representa um número negativo.

Overflow Flag (OF)

O resultado transbordou?

Ativado quando uma operação aritmética produz um resultado fora do intervalo representável (ex: somar dois números positivos e obter um negativo).

🔗
Conexão com o if: Quando a UC encontra uma instrução de desvio condicional (equivalente ao seu if), ela verifica os flags para decidir se vai ou não alterar o Program Counter. if a == b → subtrai a de b → verifica Zero Flag → se ZF=1, pula para o bloco if. Todo condicional do seu código é, no fundo, uma comparação na ULA seguida de uma verificação de flag.

Por que isso importa para você, programador?

Operadores do Python

+, -, *, //, %, &, |, ^

a & b, a | b, a ^ b em Python são diretamente as operações AND, OR e XOR da ULA. Quando você usa a % 2 == 0 para verificar paridade, você está usando a mesma lógica que a ULA usa.

Overflow em outras linguagens

Quando os números "giram"

Em C e Java, int tem 32 bits. Se você somar 2.147.483.647 + 1, o resultado é −2.147.483.648 por causa do Overflow Flag. Python evita isso com inteiros de precisão arbitrária — mas custa mais memória.

Operações bit a bit na prática

Flags, permissões, cores

Permissões de arquivo Unix (chmod 755) são bits. Cores em hex (#FF5733) são três bytes. Redes usam máscaras de bits. Entender AND/OR/XOR abre um mundo de código eficiente.

Desempenho

Shift é mais rápido que multiplicar

Compiladores inteligentes substituem x * 2 por x << 1 (shift left) automaticamente porque a ULA executa deslocamento mais rápido que multiplicação completa.

Termos-chave desta aula

ULA / ALU Unidade Lógica e Aritmética. Executa todas as operações matemáticas e lógicas da CPU.
Operandos Os valores de entrada de uma operação. A + B → A e B são os operandos.
AND / OR / XOR / NOT Operações lógicas bit a bit. Base da álgebra booleana e de toda lógica digital.
Registrador de flags Registrador que armazena propriedades do último resultado (zero, negativo, overflow, carry).
Complemento de dois Representação de números negativos em binário que permite usar o mesmo circuito de soma para subtração.
Bit shift Deslocar todos os bits para esquerda (×2) ou direita (÷2). Mais rápido que multiplicação na ULA.
