Redes de Computadores
LAN, WAN, IP, roteadores e switches
▼

Uma rede é simplesmente um conjunto de dispositivos que podem se comunicar. Mas os detalhes de como essa comunicação funciona são sofisticados — e entendê-los transforma você de usuário passivo da internet em alguém que sabe o que está acontecendo quando clica em um link.

O que é uma redeDois computadores conectados já são uma rede. LAN (rede local, como a de casa) vs. WAN (rede ampla, como a internet). Como redes se interconectam para formar a internet.
Endereço IPO "endereço postal" de cada dispositivo. IPv4 (ex: 192.168.1.1) vs. IPv6. IP público vs. privado. DHCP: como o roteador distribui endereços automaticamente.
DNS — Domain Name SystemO "livro de endereços" da internet. Como "google.com" vira um endereço IP. Hierarquia de servidores DNS. O que acontece quando um site "cai" por problema de DNS.
TCP vs UDPTCP: confiável, com confirmação de entrega (HTTP, e-mail). UDP: rápido, sem garantia (streaming, jogos online). Por que jogos online usam UDP e por que pacotes são perdidos às vezes.
Roteador e switchSwitch: conecta dispositivos dentro de uma rede local. Roteador: conecta redes diferentes (sua LAN com a internet). NAT: como vários dispositivos compartilham um único IP público.
Modelo em camadasA ideia de que a comunicação funciona em camadas (física, transporte, aplicação). Não precisa decorar o modelo OSI — precisa entender o conceito de abstração em camadas, que reaparecerá em programação.
🎯
Entregável: Você usa o comando nslookup ou dig para descobrir o IP de 5 sites conhecidos e explica por que o mesmo site pode ter IPs diferentes dependendo de onde você está no mundo.

Web, HTTP e APIs
Como uma página chega no seu navegador
▼

A web é a aplicação mais usada da internet — mas a internet é muito mais do que a web. Entender HTTP, o modelo cliente-servidor, e o que é uma API transforma você em alguém capaz de entender (e construir) qualquer aplicação conectada.

HTTP e HTTPSO protocolo da web. Request e response. Métodos: GET (buscar), POST (enviar), PUT (atualizar), DELETE (apagar). Códigos de status: 200 OK, 404 Not Found, 500 Server Error.
Cliente × ServidorModelo fundamental de aplicações modernas. O navegador (cliente) pede; o servidor responde. Onde o código roda: no navegador (frontend) ou no servidor (backend).
O que é uma APIInterface de programação — um contrato entre sistemas. REST API como o padrão mais comum. JSON como formato de dados. Você faz uma requisição real a uma API pública (ex: previsão do tempo, fatos aleatórios).
DevTools na práticaAba "Network" do Chrome/Firefox. Ver requisições HTTP ao vivo ao navegar por um site. Entender headers, body, tempo de resposta. A ferramenta favorita de todo desenvolvedor web.
🎯
Entregável: Você abre o DevTools, navega para um site (ex: Wikipedia), identifica a requisição HTML principal, uma requisição de imagem e uma de JavaScript, e explica o que cada uma faz e qual é o status code de cada uma.

Segurança Digital
Proteção, senhas e higiene digital
▼

Segurança não é um módulo separado — é uma mentalidade. Mas para desenvolver essa mentalidade, você precisa entender as ameaças reais. Esta etapa não ensina invasão, mas explica como ataques funcionam conceitualmente para que você possa escrever código mais seguro no futuro.

Senhas e autenticaçãoO que torna uma senha forte. Hashing de senhas (nunca guardar senha pura). Autenticação de dois fatores. Por que reutilizar senha é perigoso.
Criptografia — ideia geralSimétrica (uma chave para criptografar e descriptografar) vs. assimétrica (chave pública e privada). HTTPS usa criptografia. O protocolo TLS. Por que ver o cadeado verde importa.
Phishing e engenharia socialAtaques que exploram pessoas, não sistemas. Como identificar um e-mail de phishing. Por que "o elo mais fraco é humano" é verdade e o que fazer sobre isso.
Boa higiene digitalGerenciadores de senha. Updates de segurança (por que ignorá-los é arriscado). Redes Wi-Fi públicas e VPNs. Permissões de apps. O que são cookies e o que eles guardam.
🎯
Entregável: Você faz uma "auditoria de segurança" da sua própria vida digital: verifica se alguma senha foi vazada (usando haveibeenpwned.com), identifica quais apps têm permissões excessivas no celular, e cria um plano de 3 ações concretas para melhorar sua segurança.

Projeto: Glossário de Redes
Consolidar com um entregável real
▼

O projeto final da Fase 2 transforma o conhecimento absorvido em algo publicável. Você cria um mini-glossário de redes usando HTML e CSS básico — não porque precisará dominar web dev agora, mas porque o ato de escrever e publicar algo real consolida o aprendizado de uma forma que nenhum exercício escrito consegue.

HTML básico de sobrevivênciaTags fundamentais (html, head, body, h1-h3, p, ul, li, a). Não é o foco — é o veículo. O suficiente para publicar conteúdo na web.
CSS básico de sobrevivênciaCores, fontes, margens, padding. Tornar a página legível e com cara de página de verdade. Não precisa ser bonito — precisa ser funcional.
Publicar com GitHub PagesCriar um repositório no GitHub (primeiro contato com Git), fazer upload dos arquivos, e publicar o glossário com uma URL pública. Você sai com um link para compartilhar.
Apresentação oralVocê explica 5 termos do glossário de memória, usando suas próprias palavras. Este exercício de comunicação técnica será cobrado em todas as fases seguintes.
🎯
Entregável: Glossário online publicado com pelo menos 15 termos de redes definidos com suas próprias palavras, com exemplos práticos. URL pública compartilhável. Apresentação de 5 minutos.
